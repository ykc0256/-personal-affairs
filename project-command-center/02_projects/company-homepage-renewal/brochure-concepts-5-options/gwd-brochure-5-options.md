# GWD 브로슈어 5가지 방향안

## 공통 전제

GWD는 현재 보유한 수처리 설계 실행 엔진으로 설명한다. 메시지의 중심은 `AI 비전`이 아니라 `공정 조건에서 실제 설계 산출물까지 연결하는 엔지니어링 기반 기술`이다.

공통으로 사용할 수 있는 근거:

- 8년 이상 자체 개발
- 50억 원 이상 투자
- 한국, 미국, 인도 등록 / 중국 등록결정 / 유럽 심사중 등 5개국 특허 포트폴리오
- 공정 계산, 3D Layout, 도면, BOM, CAPEX/OPEX 연결
- 원자료 표현: `시뮬레이션 결과를 실제 설계 산출물로 전환하는 세계 유일의 미들웨어`
- 목표 표현: `설계 기간 최대 80% 단축`, `설계 비용 50% 절감`

---

# 1안. 설계 실행 엔진형

## 방향

현재까지 논의한 기준안이다. GWD를 `수처리 설계 실행 엔진`으로 정의하고, 공정 조건에서 3D, 도면, BOM, CAPEX/OPEX까지 이어지는 연결성을 보여준다.

## 추천 제목

`GWD 기반 하·폐수 처리장 설계 자동화`

## 한 줄 정의

공정 조건 입력부터 Mass Balance, 3D Layout, 도면, BOM, CAPEX/OPEX 산출까지 연결하는 수처리 설계 실행 엔진

## 본문 블록

| 블록 | 문구 |
| --- | --- |
| 왜 GWD인가 | 하나의 설계 조건 변경이 공정 계산, 3D, 도면, BOM, 비용 산출까지 연쇄적으로 영향을 줍니다. |
| GWD 개념 | GWD는 공정 조건에서 도면, 물량, 비용 산출까지 이어지는 수처리 설계 실행 엔진입니다. |
| 엔지니어링 가치 | 반복 수정 부담을 줄이고, 설계 검토 속도와 산출물 일관성을 높이는 기반 기술입니다. |

## 레이아웃

```text
[상단] 제목 + 한 줄 정의 + 3D Layout/플랜트 이미지
[중앙] 기존 방식 vs GWD 방식 비교
[하단] GWD 흐름도 + Proof Points
```

## 적합한 경우

- 첫 외부 공개용 시안
- GWD를 가장 안정적으로 소개해야 할 때
- 기술성과 과장 방지의 균형이 필요할 때

---

# 2안. Problem / Solution 강조형

## 방향

기존 설계 방식의 분리와 반복 수정 문제를 전면에 두고, GWD를 그 문제를 줄이는 해결 방식으로 보여준다.

## 추천 제목

`분리된 수처리 설계 업무를 하나의 흐름으로`

## 한 줄 정의

공정 계산, 3D, 도면, BOM, 비용 산출의 단절을 줄이는 GWD 설계 자동화 흐름

## 본문 블록

| 블록 | 문구 |
| --- | --- |
| 기존 문제 | 공정 계산과 도면, 물량, 비용 산출이 분리되면 조건 변경마다 반복 수정이 발생합니다. |
| GWD 방식 | GWD는 공정 조건을 기준으로 설계 산출물을 하나의 흐름 안에서 연결합니다. |
| 기대 효과 | 산출물 간 불일치를 줄이고, 설계 검토와 재작업 부담을 낮추는 방향을 제시합니다. |

## 레이아웃

```text
[상단] 문제 제기형 제목
[좌측] 기존 방식: 분리된 업무 블록
[우측] GWD 방식: 연결된 워크플로우
[하단] 반복 수정 감소 / 일관성 / 검토 속도
```

## 적합한 경우

- 비전보다 “왜 필요한지”를 먼저 설득해야 할 때
- 고객사나 내부 임원에게 문제-해결 구조로 설명할 때
- 브로슈어 첫 장에서 이해도를 높이고 싶을 때

---

# 3안. 검증 기반 / Proof 강조형

## 방향

GWD의 신뢰성을 전면에 둔다. 8년 개발, 50억+ 투자, 특허 포트폴리오 등 검증 기반을 강조한다.

## 추천 제목

`8년간 축적된 수처리 설계 자동화 기술`

## 한 줄 정의

부강테크가 자체 개발해 온 수처리 설계 실행 엔진, GWD

## 본문 블록

| 블록 | 문구 |
| --- | --- |
| 기술 기반 | GWD는 8년 이상의 자체 개발과 실제 설계 업무 적용을 통해 축적된 기반 기술입니다. |
| 투자와 특허 | 50억 원 이상 투자와 5개국 특허 포트폴리오가 설계 자동화 기반을 뒷받침합니다. |
| 연결 범위 | 공정 계산부터 3D Layout, 도면, BOM, CAPEX/OPEX까지 하나의 흐름으로 연결합니다. |

## 레이아웃

```text
[상단] 기술 축적 메시지
[중앙] Proof Points 큰 카드 3개: 8년 / 50억+ / 5개국 특허
[하단] GWD 연결 범위 다이어그램
```

## 적합한 경우

- 기술 신뢰도를 먼저 보여줘야 할 때
- 투자자, 임원, 협력사 대상 검토용
- GWD가 단순 아이디어가 아니라 이미 축적된 기술임을 강조할 때

---

# 4안. Workflow / 산출물 연결형

## 방향

GWD가 실제로 무엇을 연결하는지 흐름 중심으로 보여준다. 기능을 많이 설명하기보다 산출물 연결성을 시각화한다.

## 추천 제목

`공정 조건에서 설계 산출물까지 연결하는 GWD`

## 한 줄 정의

공정 계산 결과를 3D Layout, 도면, BOM, CAPEX/OPEX 산출로 이어주는 설계 워크플로우

## 본문 블록

| 블록 | 문구 |
| --- | --- |
| 입력 | GWD는 유입 조건과 공정 조건을 설계 흐름의 출발점으로 삼습니다. |
| 연결 | 계산 결과는 3D Layout, 도면, 물량, 비용 산출로 이어집니다. |
| 검토 | 설계 변경에 따른 산출물 검토와 갱신을 지원하는 구조를 지향합니다. |

## 레이아웃

```text
[상단] 제목 + 공정 조건 입력 이미지
[중앙] 대형 파이프라인: 입력 → 계산 → 3D → 도면/BOM → CAPEX/OPEX
[하단] 산출물 카드 5개
```

## 적합한 경우

- 이미지 시안에서 가장 직관적인 구조를 원할 때
- 실제 프로그램 화면이나 산출물 예시를 넣을 계획이 있을 때
- 홈페이지의 GWD 실제 활용 흐름과 연결할 때

---

# 5안. 미들웨어 / 차별 포지셔닝형

## 방향

GWD를 시뮬레이션 프로그램과 상세설계 CAD 도구 사이의 간극을 연결하는 수처리 특화 미들웨어로 포지셔닝한다.

## 추천 제목

`시뮬레이션과 실제 설계 산출물을 연결하는 GWD`

## 한 줄 정의

수처리 공정 시뮬레이션 결과를 실제 설계 산출물로 전환하는 설계 미들웨어

## 본문 블록

| 블록 | 문구 |
| --- | --- |
| 시장 간극 | 시뮬레이션은 성능 검토에 강하지만, 실제 설계 산출물 전환에는 한계가 있습니다. |
| GWD 위치 | GWD는 공정 계산과 실제 도면, 물량, 비용 산출 사이를 연결합니다. |
| 차별성 | 수처리 도메인 지식과 설계 로직이 결합된 설계 미들웨어를 지향합니다. |

## 레이아웃

```text
[상단] 미들웨어 포지셔닝 제목
[중앙] 시뮬레이션 → GWD → 실제 설계 산출물
[하단] 세계 유일 미들웨어 / 8년 / 50억+ / 특허
```

## 적합한 경우

- GWD의 차별성을 강하게 보여주고 싶을 때
- ESAIN, Transcend 등 비교 맥락을 내부적으로 참고할 때
- “왜 GWD가 별도 기술로 필요한가”를 설명할 때
