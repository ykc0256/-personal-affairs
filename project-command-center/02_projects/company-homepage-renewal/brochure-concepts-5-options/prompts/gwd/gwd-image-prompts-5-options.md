# GWD 브로슈어 5가지 안별 이미지 생성 프롬프트

## 공통 제한

모든 프롬프트에 공통 적용한다.

- Create ONE A4 vertical Korean corporate brochure page.
- External public brochure.
- Korean corporate engineering style.
- Dark navy header, white/light gray body, cyan/teal/blue accents.
- Use realistic water treatment facility, 3D plant layout, engineering workflow, thin-line icons.
- No cartoon, no generic AI robot, no decorative blobs, no purple-heavy gradient.
- Do not include WAI or AGWD as the main topic.
- Do not include TRL 5 or any TRL score.
- Text should be readable Korean, not tiny placeholder text.
- Text/visual balance: about 35% text and 65% diagrams/visual structure.

# 1안 프롬프트 — 설계 실행 엔진형

```text
Create ONE A4 vertical Korean corporate brochure page for GWD only.

Title:
"GWD 기반 하·폐수 처리장 설계 자동화"

Subtitle:
"공정 조건 입력부터 Mass Balance, 3D Layout, 도면, BOM, CAPEX/OPEX 산출까지 연결하는 수처리 설계 실행 엔진"

Main concept:
GWD is a water treatment design execution engine connecting process conditions, Mass Balance, 3D Layout, drawings, BOM, and CAPEX/OPEX.

Layout:
- Top hero: title, subtitle, realistic 3D water treatment plant or technical plant layout.
- Middle: comparison diagram titled "기존 방식 vs GWD 방식".
- Left: disconnected blocks: 공정 계산, 3D 설계, 도면, BOM, 비용 산출.
- Right: connected workflow: 공정 조건 입력 → 공정 계산 → 3D Layout → 도면/BOM → CAPEX/OPEX.
- Bottom: three proof cards: 8년+ 자체 개발, 50억+ 투자, 5개국 특허.

Short text blocks:
- 왜 GWD인가: "하나의 설계 조건 변경이 공정 계산, 3D, 도면, BOM, 비용 산출까지 연쇄적으로 영향을 줍니다."
- GWD 개념: "GWD는 공정 조건에서 도면, 물량, 비용 산출까지 이어지는 수처리 설계 실행 엔진입니다."
- 엔지니어링 가치: "반복 수정 부담을 줄이고, 설계 검토 속도와 산출물 일관성을 높이는 기반 기술입니다."

Style:
Professional environmental engineering brochure, technical, credible, executive-friendly.
```

# 2안 프롬프트 — Problem / Solution 강조형

```text
Create ONE A4 vertical Korean corporate brochure page for GWD only.

Title:
"분리된 수처리 설계 업무를 하나의 흐름으로"

Subtitle:
"공정 계산, 3D, 도면, BOM, 비용 산출의 단절을 줄이는 GWD 설계 자동화 흐름"

Main concept:
Show the pain of fragmented design work and present GWD as the connected engineering workflow.

- Top: problem-driven title and subtitle.
- Center split layout: left "기존 방식", right "GWD 방식".
- Existing method visual: fragmented work blocks, broken lines, repeated revision markers.
- GWD method visual: one connected data-flow line from 공정 조건 to CAPEX/OPEX.
- Bottom: three value chips: 반복 수정 감소, 산출물 일관성, 검토 속도 개선.

Short text blocks:
- 기존 문제: "공정 계산과 도면, 물량, 비용 산출이 분리되면 조건 변경마다 반복 수정이 발생합니다."

- 기대 효과: "산출물 간 불일치를 줄이고, 설계 검토와 재작업 부담을 낮추는 방향을 제시합니다."

Clean B2B engineering brochure, clear problem-solution visual contrast, restrained but polished.
```

