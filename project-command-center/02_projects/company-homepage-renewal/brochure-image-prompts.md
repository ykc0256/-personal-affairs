# GWD / WAI 브로슈어 이미지 생성 프롬프트

## 사용 목적

이 문서는 GWD와 WAI 1장 브로슈어 시안을 이미지로 생성하기 위한 프롬프트 모음이다.

- `GWD 브로슈어`: 현재 보유한 수처리 설계 실행 엔진을 설명하는 1장짜리 기술 브로슈어
- `WAI 브로슈어`: AGWD 과제 세부 내용이 아니라, 수처리 시설의 설계·시공·운영 데이터를 생애주기 관점으로 연결하고 환류시키는 WATER AI 원론을 설명하는 1장짜리 브로슈어

먼저 GWD 시안을 생성해 전체 톤을 잡고, 이후 같은 디자인 시스템으로 WAI 시안을 생성하는 순서를 권장한다.

# 1. GWD 브로슈어 이미지 생성 프롬프트

```text
Create a professional one-page Korean corporate brochure design for "GWD 기반 하·폐수 처리장 설계 자동화".

Format:
- A4 vertical one-page brochure.
- Clean B2B engineering style.
- Polished, technical, and executive-friendly.
- It should look like a brochure for an environmental engineering company, not a generic SaaS landing page.

Primary message:
GWD is a water treatment design execution engine that connects process conditions, Mass Balance, 3D Layout, drawings, BOM, and CAPEX/OPEX into one engineering workflow.

Main visual direction:
- Dark navy and deep charcoal header area.
- White or very light gray body area for readability.
- Accent colors: cyan, teal, blue, and small controlled green accents.
- Include subtle water treatment plant, environmental engineering, 3D plant layout, digital twin, and engineering grid imagery.
- Use structured engineering diagrams, thin lines, clean icons, and data-flow elements.
- Avoid purple-heavy gradients, decorative blobs, playful illustrations, cartoon style, AI robot imagery, and overly abstract futuristic effects.

Top hero section:
- Large Korean title: "GWD 기반 하·폐수 처리장 설계 자동화"
- Subtitle: "공정 조건 입력부터 Mass Balance, 3D Layout, 도면, BOM, CAPEX/OPEX 산출까지 연결하는 수처리 설계 실행 엔진"
- Place a realistic 3D water treatment facility, plant layout, or technical 3D engineering model visual on the right side.
- The top area should immediately communicate water treatment engineering and design automation.

Explanatory text density:
- This should feel like a real corporate technical brochure, not just a workflow poster.
- Include 2-3 very short Korean text blocks with actual explanatory copy.
- Each text block should be only 1-2 short lines.
- Keep the tone technical, credible, and grounded in current capability.

Problem / why GWD section:
Add a text block titled "왜 GWD인가".
Use this Korean copy, adjusted only for layout if needed:
"하나의 설계 조건 변경이 공정 계산, 3D, 도면, BOM, 비용 산출까지 연쇄적으로 영향을 줍니다."

GWD concept section:
Add a text block titled "GWD 개념".
Use this Korean copy, adjusted only for layout if needed:
"GWD는 공정 조건에서 도면, 물량, 비용 산출까지 이어지는 수처리 설계 실행 엔진입니다."

Engineering value section:
Add a text block titled "엔지니어링 가치".
Use this Korean copy, adjusted only for layout if needed:
"반복 수정 부담을 줄이고, 설계 검토 속도와 산출물 일관성을 높이는 기반 기술입니다."

Middle section:
Create a clear comparison area titled "기존 방식 vs GWD 방식".

Left side: "기존 방식"
Show disconnected process blocks:
- 공정 계산
- 3D 설계
- 도면
- BOM
- 비용 산출
These should appear as separated boxes with broken lines or disconnected workflow indicators, showing fragmented work and repeated manual revision.

Right side: "GWD 방식"
Show a connected workflow:
공정 조건 입력 → 공정 계산 → 3D Layout → 도면/BOM → CAPEX/OPEX
Use arrows, clean engineering icons, and connected data-flow lines.
The GWD side should feel more integrated, structured, and reliable than the existing method.

Lower section:
Add a "Proof Points" area with three metric cards:
- 8년+ 자체 개발
- 50억+ 투자
- 5개국 특허

Add a compact capability row:
- 공정 계산 자동화
- 설비·구조물 규격 산정
- 3D Layout 생성
- 설계 도면 생성
- 물량·비용 산출

Bottom message:
"GWD는 수처리 설계 업무의 반복 수정과 산출물 불일치를 줄이고, 설계 검토 속도와 일관성을 높이는 기반 엔진입니다."

Typography:
- Use clean Korean sans-serif typography.
- Strong hierarchy with a large title, clear section headings, and concise body text.
- All Korean text must be legible and professionally typeset.
- Keep the layout dense enough for a technical brochure but still easy to scan.
- Use short paragraph blocks, concise labels, diagram text, and compact metric cards.
- Balance text and diagrams at about 35% text and 65% visual structure.

Overall mood:
Reliable, technical, infrastructure-focused, engineering-grade, and credible.
```

```text
- GWD: design data foundation
```

# 4. GWD / WAI 2개 이미지 확실히 생성하는 방식

이미지 생성기가 한 프롬프트에서 계속 한 장만 만들거나, 두 브로슈어를 하나의 포스터로 합쳐버릴 경우에는 **한 번에 2장을 만들라고 요청하지 않는다.**

가장 확실한 방식은 아래처럼 **프롬프트를 2번 실행**하는 것이다.

1. 먼저 `GWD 이미지 1개 생성 프롬프트`를 실행한다.
2. 그 다음 `WAI 이미지 1개 생성 프롬프트`를 별도로 실행한다.

이 방식으로 해야 결과물이 실제로 `GWD 브로슈어 이미지 1개`, `WAI 브로슈어 이미지 1개`로 분리된다.

# 7. 사용 순서 제안

1. `4-1. GWD 이미지 1개 생성 프롬프트`를 실행한다.
2. 결과물로 GWD 브로슈어 이미지 1개를 저장한다.
3. `4-2. WAI 이미지 1개 생성 프롬프트`를 별도로 실행한다.
4. 결과물로 WAI 브로슈어 이미지 1개를 저장한다.
5. 이렇게 해야 실제로 이미지 2개가 나온다.
6. 두 장을 한 화면에서 비교하고 싶을 때만 `5. 그래도 한 번에 2개를 시도하고 싶을 때 쓰는 프롬프트`를 사용한다.
7. 생성 이미지에서 텍스트가 깨지거나 어색하면, 실제 텍스트는 디자인 툴에서 별도 편집하는 방식으로 진행한다.
