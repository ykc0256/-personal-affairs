# GWD / WAI 브로슈어 자료 정리

## 현재 상태

2026-05-26 기준으로 GWD와 WAI를 각각 1장짜리 브로슈어로 분리 작성하는 방향을 잡았다.

- `GWD`: 현재 보유한 수처리 설계 실행 엔진 소개
- `WAI`: GWD → AGWD → WAI 흐름 속에서 수처리 시설의 설계·시공·운영 데이터를 생애주기 관점으로 연결하고 환류시키는 WATER AI 원론 소개

브로슈어 작성의 1순위 기준 자료는 과제 발표자료와 과제 제안서다. 다만 WAI 브로슈어는 AGWD 세부 모듈이나 플랫폼 확장 계획보다 WATER AI의 원론적 개념, 설계 데이터의 중요성, 생애주기 데이터 환류 구조를 우선한다. AGWD는 GWD와 WAI 사이의 발전 단계로 언급할 수 있고, 오믈렛·세부 AI 모듈명·협력사 수행 체계는 외부 공개용 브로슈어에서 제외한다.

## 4. 브로슈어 메시지 구조

### GWD 브로슈어

GWD는 현재 보유한 설계 실행 엔진으로 설명한다.

핵심 흐름:

```text
설계 단계 AI 공백
→ 기존 설계 방식의 분리와 반복 수정
→ GWD 설계 실행 엔진
→ 공정 계산 / 3D Layout / 도면 / BOM / CAPEX·OPEX 연결
→ 8년 개발, 50억+ 투자, 특허 등 Proof Points
```

강조할 문장:

> GWD는 수처리 공정 조건과 설계 로직을 연결해 Mass Balance, 3D Layout, 도면, BOM, CAPEX/OPEX 산출을 하나의 설계 흐름으로 구성하는 설계 실행 엔진입니다.

본문 설명 블록:

| 블록 | 핵심 내용 |
| --- | --- |
| 왜 GWD인가 | 조건 변경이 공정 계산, 3D, 도면, BOM, 비용 산출에 연쇄 영향을 준다. |
| GWD 개념 | 공정 조건부터 설계 산출물까지 연결하는 설계 실행 엔진이다. |
| 엔지니어링 가치 | 반복 수정 부담을 줄이고 산출물 일관성을 높이는 기반 기술이다. |

### WAI 브로슈어

WAI는 수처리 시설의 성능과 운영 효율이 운영 단계에서만 결정되지 않는다는 문제의식에서 출발한다. 설계 단계의 공정 조건, 설비 구성, 배치, 물량, 비용 데이터가 시공과 운영을 거쳐 다시 설계 기준으로 돌아오는 WATER AI의 원론적 환류 구조로 설명한다.

GWD, AGWD, WAI의 관계는 다음처럼 정리한다.

```text
GWD: 수처리 설계 데이터를 구조화하는 기반 엔진
→ AGWD: GWD 기반 설계 자동화를 지능형 설계 자동화 과제로 고도화
→ WAI: 설계·시공·운영·재설계 데이터를 환류시키는 WATER AI 개념
```

핵심 흐름:

```text
설계 데이터 구조화
→ 시공 데이터 연결
→ 운영 데이터 축적
→ 재설계와 설계 기준 개선으로 환류
→ WATER AI 개념
```

강조할 문장:

> WAI는 수처리 시설의 설계, 시공, 운영 과정에서 발생하는 데이터를 생애주기 관점으로 연결하고, 축적된 데이터를 다시 설계 기준과 운영 개선으로 환류시키는 WATER AI 개념입니다.

본문 설명 블록:

| 블록 | 핵심 내용 |
| --- | --- |
| 왜 WAI인가 | 수처리 AI는 운영 데이터만이 아니라 설계 데이터에서 시작된다. |
| WAI 개념 | 설계·시공·운영 데이터를 생애주기 흐름으로 연결하는 WATER AI 개념이다. |
| GWD / AGWD / WAI 관계 | GWD → AGWD → WAI로 이어지는 데이터 환류 방향이다. |

본문 밀도 기준:

- 실제 브로슈어는 기능 라벨과 다이어그램만으로 구성하지 않되, 긴 본문도 피한다.
- GWD와 WAI 모두 2~3개의 짧은 본문 설명 블록을 포함한다.
- 각 본문 블록은 1~2줄 수준으로 유지한다.
- WAI는 개발 중인 개념이므로 완료형 제품 설명보다 문제의식, 개념 정의, 환류 방향을 중심으로 작성한다.

## 5. 표현 주의사항

아래 표현은 외부 공개용 브로슈어에서 사용하지 않거나, 원자료 표현 여부 확인 후 사용한다.

- `TRL 8 달성`
- `TRL 5`
- `18개월 상용화 완료`
- `100억 매출 확정`
- `1일 내 재생성`
- `모든 수처리 공정 자동 설계`
- `즉시 SaaS 제공 가능`
- `AI가 설계·시공·운영을 모두 자동화`

권장 표현:

- GWD(Generative WWTP Design)
- 검증된 GWD 설계 엔진
- 8년간 축적된 설계 자동화 기술
- 50억+ 투자 기반
- 특허 기반 설계 자동화 기술
- 시뮬레이션 결과를 실제 설계 산출물로 전환하는 세계 유일의 미들웨어
- 설계 기간 최대 80% 단축, 설계 비용 50% 절감
- 실제 설계업무 적용 기반
- 공정 계산, 3D Layout, 도면, BOM, CAPEX/OPEX 연결
- 설계 데이터의 시공·운영 연결 기반
- 운영 데이터의 재설계 환류
- 설계 기준 개선
- WaterAI 개념 기반의 생애주기 데이터 환류
