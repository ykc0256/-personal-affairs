# 홈페이지 구성 논의 정리

작성일: 2026-05-14

## 1. 요청 배경

- 회사 홈페이지 리뉴얼과 관련해 1~3Page를 어떻게 구성할지 개략적으로 정리할 필요가 있었음
- 강수진 리더와 협의할 수 있도록 상세 보고서가 아니라 짧은 구성 정리 형태가 필요했음
- 사용자가 제공한 `홈페이지 정리 예시.png`는 타팀 계획 예시이며, 해당 이미지처럼 간략한 항목 중심 문서 형식을 참고함

## 2. 예시 이미지 처리

- `홈페이지 정리 예시.png`의 텍스트를 확인함
- 이미지 내용은 별도 Markdown 파일로 전사함
- 생성 파일: `홈페이지 정리 예시.md`

## 3. 홈페이지 구성 보고서 작성 및 수정

- 최초에는 전체 랜딩페이지 IA 중심의 상세 보고서가 작성되었으나, 사용자의 의도와 맞지 않아 수정함
- 이후 1~3Page 중심의 개략보고서로 재작성함
- 최종적으로 예시 이미지와 유사하게 간략한 항목 정리 형식으로 변경함

현재 보고서 파일:

- `02_projects/company-homepage-structure-report.md`

## 4. 최종 구성 방향

### 1Page: Problem / Solution

- 기존 하수처리 설계 업무의 문제점과 GWD 기반 해결 방향 반영
- 기존 방식: 공정 계산, 3D 설계, 도면, 물량, 비용 산출이 각각 분리됨
- GWD 방식: 공정 조건에서 시작해 3D Layout, 도면, 물량, CAPEX/OPEX 산출까지 연결
- 핵심 메시지: 반복 수정 감소, 산출물 일관성 향상, 설계 검토 속도 개선

### 2Page: WATER AI 환류 구조

- 초기에 “하류 구조”라고 잘못 표현했으나, 사용자가 “환류 구조”라고 정정함
- 이후 문서 전체에서 `하류 구조` 표현을 `환류 구조`로 수정함
- 의미는 설계 데이터가 시공, 운영, 재설계를 거쳐 다시 설계 기준 개선으로 돌아오는 순환 구조임

### 3Page: GWD / WAI Planning

- GWD와 WAI Planning의 실제 활용 영역 반영
- 영상 업로드 또는 기존 프로그램 화면이 들어갈 수 있는 구역 필요
- 구성 예시: 공정 조건 입력 → 계산 → 3D Layout → 도면/물량/비용 산출 → 결과 검토
- 기능 나열보다 실제 사용 흐름 중심으로 정리하는 방향

## 5. 최종 문서 구조

사용자 요청에 따라 보고서 구조는 다음처럼 단순화함.

1. 홈페이지 1~3Page 구성 방향
2. 단기 업무 분장

삭제된 항목:

- 추가 확인 필요 항목
- 강수진 리더 협의 필요 사항

## 6. 단기 업무 분장 정리

최종 업무 분장은 다음과 같이 정리함.

- 최윤규 리더: 1~3Page 구성 초안 및 문구 정리, 기술소개자료, 브로슈어, 이미지/영상 자료 확인 및 제공

변경 이력:

- 처음에는 최윤규 리더, 강수진 리더, 이옥주 리더로 나눴음
- 이후 이옥주 리더 항목 삭제
- 이후 강수진 리더 항목도 삭제
- 강수진 리더가 맡는 것으로 적었던 업무는 모두 최윤규 리더 업무로 통합
