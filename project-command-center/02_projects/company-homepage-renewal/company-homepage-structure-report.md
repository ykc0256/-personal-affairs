# 홈페이지 구성 정리

## 1. 홈페이지 1~3Page 구성 방향

### 1.1 1Page: Problem / Solution

- 기존 하수처리 설계 업무의 문제점과 GWD 기반 해결 방향 반영
- 기존 방식: 공정 계산, 3D 설계, 도면, 물량, 비용 산출이 각각 분리됨
- GWD 방식: 공정 조건에서 시작해 3D Layout, 도면, 물량, CAPEX/OPEX 산출까지 연결
- 핵심 메시지: 반복 수정 감소, 산출물 일관성 향상, 설계 검토 속도 개선

※ GWD가 단순 AI 이미지/컨셉이 아니라 실제 설계 자동화 엔진이라는 점이 드러나야 함
※ 과제 발표자료를 참고하여 수정 보완 예정

### 1.2 2Page: WATER AI 환류 구조

- GWD에서 생성된 설계 데이터가 시공, 운영, 재설계로 다시 돌아오는 환류 구조 반영
- 설계 데이터 → 시공 정보 → 운영 데이터 → 재설계 기준 개선 흐름으로 구성
- WATER AI는 현재 완성형 제품이라기보다, GWD 기반 데이터를 확장하는 플랫폼 방향으로 정리
- 기존 `WATER AI CYCLE` 참고 이미지 또는 유사한 순환 구조 다이어그램 필요
- 개발에 사용된 금액/개발시간/특허 부분에 대해 내용 추가

### 1.3 3Page: GWD / WAI Planning

- GWD와 WAI Planning의 실제 활용 영역 반영
- 영상 업로드 또는 기존 프로그램 화면이 들어갈 수 있는 구역 필요
- 구성 예시: 공정 조건 입력 → 계산 → 3D Layout → 도면/물량/비용 산출 → 결과 검토
- 사용자가 WAI Planning에서 무엇을 확인할 수 있는지 간략히 보여주는 방향

※ 기능을 너무 많이 나열하기보다 실제 사용 흐름 중심으로 정리 필요
※ 기본 구조 및 WAI Planning에 대해 상세한 논의 필요
