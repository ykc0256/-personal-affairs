## 1. GWD vs ESAIN 자동화 위치 비교

| 구분 | GWD | ESAIN / ESAPRO |
|---|---|---|
| 자동화 초점 | **기본설계 / 계산 자동화** | **상세설계 / 문서·물량 산출 자동화** |
| 핵심 영역 | 수처리 공정 계산 + 규격 sizing + 도면/3D/BOM 생성 | CAD 기반 P&ID·3D Piping·Electrical 연계 및 물량 산출 |
| 제품 구조 | Unity 기반 PC 설치형 + 관리자 Web (자체 플랫폼) | AutoCAD/BricsCAD 기반 CAD 확장형 솔루션 |
| 공정 설계 | O (계산 자동화가 핵심) | X |
| AI 도입 | 초격차 전략의 핵심 축 | 발표된 AI 도입 계획 없음 |

- ESAIN 케이스 히스토리에 Water & Waste Treatment 4건 존재 → 고객군 일부 겹치나 공정 의사결정 영역은 중복 없음
- ESAIN의 실질 경쟁 영역 = Autodesk Plant 3D (상세설계 CAD 생태계)

---

## 2. GWD 타겟 / 포지셔닝 방향

**두 축 동시 추진**

| 축 | 방향 | 내용 |
|---|---|---|
| 도메인 집중 | BKT 역량 극대화 | 수처리 도메인·공정 설계 깊이가 핵심 차별성. 현재 공정 3~4개에서 도메인 폭 확장 |
| 상세설계 지향 | 영역 확장 | 최종 목표: 상세설계까지 자동화 → 엔지니어링 펌 전환. 실시설계 없으면 환류 흐름이 끊김 |

→ 두 축이 결합될 때 **도메인 기반 설계 자동화의 차별적 포지션**이 완성됨

---

## 3. 경쟁 환경

- **수처리 설계 도메인 직접 경쟁사: Transcend가 사실상 유일**
- ESAIN + Transcend 협력 시나리오 — 도면·물량 산출 + 도메인 자동화 결합으로 강력한 시너지 가능 → **모니터링 필수**

---

## 4. AI 활용 및 암묵지 학습 전략

- **초격차 전략**: 특허는 AI가 회피 자동화 → 보호 수단으로 한계. 진짜 무기 = **속도 + 데이터 독점**
- **암묵지 확보**: 설계사·실무자 인터뷰 → 공종별 암묵지 수집 → 데이터화 → AI 학습 → 상세설계 확장 데이터 자체 확보
- **보안 정책 수립 필수**: AI 활용 시 경쟁사 빠른 추격 가능 → 데이터 소스 보호 필요
- **우려**: 단기간 내 AI·수처리·상세설계 도메인 동시 소화 부담. 팀 내 도메인 검토 역량 부족 가능성
- **보완**: 테스트 에이전트 + 외부 기술사 용역 + 사용자 피드백 다층 구조

---

## 5. 협업 vs 자체 개발

- ESAIN 협업 가능성 검토 → **결론: 종속 위험** (Maven·SolidWorks 사례 참고). 자체 개발 추진
- BKT 도메인 외 영역(구조 검토·스트레스 해석 등)은 **외부 API 연동**으로 해결
- ESAIN은 직접 협력 대상 아닌 **레퍼런스·분석 대상**으로 유효한 기능만 GWD에 적용

---

## 6. GWD 신뢰도 확보 방안

- BioWin, GPS-X, Sumo AI 등 시장 검증된 시뮬레이션 엔진 **연동** → GWD 계산 결과 신뢰도 확보

---

## 브로셔/홈페이지 메시지 활용 포인트

1. **경쟁 포지셔닝 메시지**: "ESAIN·Autodesk 같은 상세설계 CAD 툴과 달리, GWD는 공정 조건 입력만으로 설계 산출물을 생성하는 기본설계 자동화 엔진"
2. **차별점 강조**: 수처리 도메인 특화 + 공정 계산 자동화 = Transcend 외 실질 경쟁자 없음
3. **AGWD 비전**: 기본설계 → 상세설계까지 자동화 확장 → 엔지니어링 펌 수준의 설계 자동화 플랫폼
4. **신뢰도 메시지**: BioWin·GPS-X 연동 기반 계산 결과 검증 → 단순 AI 추정이 아닌 검증된 엔진 기반
