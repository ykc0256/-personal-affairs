# WAI 브로슈어 이미지 생성 프롬프트

아래 프롬프트만 복사해서 이미지 생성 도구에 붙여넣는다.

이 프롬프트는 **WAI 브로슈어 이미지 1개만** 생성하기 위한 것이다.  
GWD 브로슈어는 별도 파일 `gwd-brochure-image-prompt.md`를 사용한다.

---

```text
Create ONE image only.
Create ONE A4 vertical Korean corporate brochure page.
This image is for the WAI brochure only.

Do not include a separate GWD brochure page.
Do not create a GWD brochure.
Do not create two pages.
Do not create a two-page spread.
Do not create a combined poster.

Title:
"WAI, 수처리 시설의 생애주기를 연결하는 WATER AI"

Subtitle:
"설계, 시공, 운영 데이터를 생애주기 관점으로 연결하고 축적된 데이터를 다시 설계 기준과 운영 개선으로 환류시키는 WATER AI 개념"

Main concept:
WAI is a WATER AI concept that explains why water infrastructure AI should start from structured design data, connect design, construction, and operation data, and return operational experience to redesign and improved design standards.

Format:
- One A4 vertical brochure page.
- Korean corporate engineering concept brochure.
- Strategic, technical, credible, and future-oriented.
- It should look like an environmental engineering lifecycle data intelligence brochure.
- It should be suitable for a government project presentation, executive review, and technical partner meeting.

Visual style:
- Dark navy header.
- White or very light gray body.
- Cyan, teal, blue, and restrained green accents.
- Water infrastructure lifecycle data visuals.
- Circular lifecycle data flow.
- Technical data nodes, thin-line icons, clean diagrams, and lifecycle data-flow blocks.
- Clean and structured grid layout.
- No cartoon.
- No generic AI robot.
- No decorative blobs.
- No purple-heavy gradient.
- No playful illustration style.

Page layout:

Top hero section:
- Large Korean title: "WAI, 수처리 시설의 생애주기를 연결하는 WATER AI"
- Subtitle under the title.
- Right side visual: connected water treatment plant data, dashboard overlays, and lifecycle data flow.
- The hero should communicate lifecycle data feedback, design-to-operation continuity, and water infrastructure intelligence.

Explanatory text density:
- This should feel like a real corporate brochure, not just a diagram poster.
- Include 2-3 very short Korean text blocks with actual explanatory copy.
- Each text block should be only 1-2 short lines.
- Keep the tone conceptual and directional because WAI is under development.
- Avoid completed-product wording such as "제공합니다", "완료했습니다", "즉시 사용 가능합니다".
- Avoid long paragraphs. This should still feel like a brochure, not a report.

Problem / why WAI section:
Add a text block titled "왜 WAI인가".
Use this Korean copy, adjusted only for layout if needed:
"수처리 시설의 성능은 운영 단계만이 아니라 초기 설계 데이터에서 시작됩니다."

WAI concept section:
Add a text block titled "WAI 개념".
Use this Korean copy, adjusted only for layout if needed:
"WAI는 설계·시공·운영 데이터를 생애주기 흐름으로 연결하는 WATER AI 개념입니다."

GWD / AGWD / WAI relationship section:
Add a compact text block near the relationship diagram:
"GWD는 설계 데이터 기반, AGWD는 지능형 설계 자동화, WAI는 생애주기 데이터 환류 개념입니다."

Main structure section:
Show a simple lifecycle data feedback diagram:
설계 데이터 → 시공 데이터 → 운영 데이터 → 재설계 기준

Under each stage:
- 설계 데이터: 공정, 설비, 3D, 도면, 물량, 비용
- 시공 데이터: 자재, 배치, 시공 이력
- 운영 데이터: 수질, 에너지, 성능, 유지관리
- 재설계 기준: 운영 결과를 다음 설계 기준으로 반영

This section should make it clear that WAI is the big-picture WATER AI lifecycle feedback concept. AGWD may be mentioned only as a transition step between GWD and WAI, not as the main topic and not as detailed module explanation.

Central cycle section:
Create a WATER AI CYCLE circular diagram.
Put "WATER AI" or "WAI" at the center.

Cycle labels:
- 설계 Engineering
- 조달·시공 Procurement & Construction
- 운영 Operation
- 재설계 Re-Design
- 설계 기준 개선

Use circular arrows and connected data nodes.
The diagram should show that operation data returns to redesign and improves future design standards.

Core concept section:
Show four compact concept cards:
- 데이터 구조화
- 데이터 연결
- 데이터 축적
- 데이터 환류

These cards should explain the WATER AI concept at a high level. Do not show detailed AGWD modules.

Relationship note:
If the brochure includes the relationship among systems, show it simply as:
GWD → AGWD → WAI
- GWD: design data foundation
- AGWD: intelligent design automation advancement
- WAI: WATER AI lifecycle feedback concept

Lower principle section:
Add principle labels:
- 설계 데이터 구조화
- 생애주기 데이터 연결
- 운영 경험 축적
- 설계 기준 개선
- WATER AI 환류 구조

Bottom message:
"WAI는 설계, 시공, 운영 데이터를 생애주기 관점으로 연결하고, 운영 결과를 다시 설계 기준과 운영 개선으로 환류시키는 WATER AI 개념입니다."

Typography:
- Clean Korean sans-serif typography.
- Large readable Korean text.
- Strong visual hierarchy.
- Use short paragraph blocks, labels, modules, compact captions, and diagrams.
- Do not use tiny placeholder text. Korean copy should be readable.
- Balance text and diagrams at about 35% text and 65% visual structure.
- Make all Korean text professional and legible.

Overall mood:
Strategic, technical, data-driven, credible, and platform-oriented.

Content restrictions:
- AGWD may be mentioned only as a brief transition step: GWD → AGWD → WAI.
- Do not mention Omlet or any external partner.
- Do not show AI Decision Layer module names.
- Do not make the page look like a government project task breakdown.
- Do not make WAI look like a completed commercial AI operations platform.
- Do not overemphasize platform expansion; emphasize lifecycle data feedback and design data intelligence.
- Do not include TRL 5 or any TRL score.
```
