# Notion 수정사항 작업 지시서

## 작업 목적

현재 Notion 문서의 `전기계산서 생성`, `전기 도면 생성`, `3D layout`, `산출물 & 내역서` 관련 내용을 정리하고 보완한다.  
각 항목은 AI 또는 작업자가 바로 수정할 수 있도록 구조, 삭제/변경 대상, 추가해야 할 설명을 명확히 반영한다.

---

## 1. 전기계산서 생성

### 1.1 상세 구조 항목 정리

- `전기계산서 상세 구조`의 1~10번 항목 중 기존 `9번` 항목을 삭제한다.
- 기존 `10번 참고` 항목은 `9번 참고`로 번호를 변경한다.

### 1.2 참고 항목 구조 변경

- 변경된 `9번 참고` 항목에 포함된 내용 중 `(참조)`와 관련된 내용을 정리한다.
- `(참조)`라는 문구는 삭제한 형태로 작성한다.
- 관련 내용은 별도의 상위 제목을 생성한 뒤, 테이블 형식으로 재구성한다.
- 테이블에는 항목명, 설명, 적용 위치, 비고 등을 포함해 문서에서 바로 이해할 수 있도록 구성한다.

### 1.3 DB 기준정보 공통 필드 정리

- `DB 기준정보 공통 필드`의 하위 내용들도 테이블 형태로 변경한다.
- 각 필드는 필드명, 설명, 데이터 유형, 사용 목적, 비고 등의 열을 기준으로 정리한다.

### 1.4 자동 생성 산출물 샘플 이미지 영역 추가

- 자동 생성 산출물 영역에서 `동력설비 부하계산서` 하위에 샘플 이미지를 삽입할 수 있는 구조를 추가한다.
- 자동 생성 산출물 영역에서 `부하일람표` 하위에 샘플 이미지를 삽입할 수 있는 구조를 추가한다.
- 샘플 이미지 영역은 실제 이미지가 없어도 추후 이미지를 넣을 수 있도록 명확한 placeholder 형태로 구성한다.

### 1.5 문서 전체 검수

- `전기계산서 생성` 전체 문서에서 오탈자, 어색한 문장, 의미가 불명확한 표현을 수정한다.
- 설명이 부족하거나 흐름상 이해하기 어려운 부분은 보완 문장을 추가한다.
- 기존 문서의 의도와 범위는 유지하되, 작업자가 바로 이해할 수 있는 문장으로 정리한다.

## 2. 전기 도면 생성

### 2.1 생성 가능 도면 종류 구조 변경

- `생성 가능 도면 종류`에 포함된 각 세부 항목 아래에 샘플 이미지를 삽입할 수 있는 구조를 추가한다.
- 각 도면 종류별로 샘플 이미지 영역을 분리하여, 어떤 이미지가 어떤 도면에 해당하는지 명확히 구분되도록 구성한다.
- 실제 이미지가 없는 경우에도 추후 이미지를 넣을 수 있도록 placeholder를 추가한다.

### 2.2 케이블스케줄 샘플 이미지 구조 변경

- `케이블스케줄` 관련 3종 항목은 각각 별도 이미지를 넣는 구조가 아니라, 3종을 대표하는 1개의 샘플 이미지를 넣을 수 있도록 변경한다.
- 케이블스케줄 3종이 하나의 샘플 이미지를 공유한다는 점을 문서에 명확히 설명한다.

## 3. 3D layout

### 3.1 전기 트레이 모드 세부 흐름 보완

- `전기 트레이 모드`의 세부 흐름 중 마지막 단계에 다음 내용을 추가한다.
- 전선 연결 이후 산출된 길이값이 `동력부하계산서`와 연결된다는 점을 명시한다.
- 단순 설명이 아니라, 전체 흐름에서 어떤 값이 어디로 전달되는지 이해할 수 있도록 작성한다.

예시 문장:

> 전선 연결이 완료되면 트레이 및 케이블 경로 기준으로 길이값이 산출되며, 해당 길이값은 동력부하계산서 작성 및 산출 로직과 연계된다.

## 4. 산출물 & 내역서

### 4.1 내역서 수량 산출 로직 항목 추가

- `산출물 탭 구조`와 `내역서 Excel 시트 구조` 사이에 `내역서 수량 산출 로직` 항목을 새로 추가한다.
- 해당 항목은 산출물에서 내역서로 이어지는 중간 로직을 설명하는 역할을 해야 한다.
- 산출물 데이터가 어떤 기준으로 내역서 수량으로 변환되는지 단계별로 작성한다.

### 4.2 내역서 수량 산출 로직 상세화

- `내역서 수량 산출 로직`은 가능한 한 상세하게 작성한다.
- DB 기준정보와 어떻게 연계되는지 설명한다.
- 산출물 데이터, DB 기준정보, 내역서 수량 간의 관계를 명확히 작성한다.
- 필요 시 레퍼런스 또는 참고 기준을 추가한다.
- 로직 설명에는 입력 데이터, 처리 기준, 산출 결과, 예외 처리 기준 등을 포함한다.

권장 구성:

| 구분 | 설명 |
| --- | --- |
| 입력 데이터 | 산출물 또는 3D layout에서 전달되는 원천 데이터 |
| DB 연계 기준 | 기준정보 DB에서 참조하는 필드 및 매핑 조건 |
| 수량 산출 방식 | 수량을 계산하거나 집계하는 기준 |
| 산출 결과 | 내역서 Excel 시트에 반영되는 결과값 |
| 예외 처리 | 누락 데이터, 미매핑 데이터, 기준값 부재 시 처리 방식 |
| 레퍼런스 | 참고 문서, 기준표, 기존 산출 방식 등 |

### 4.3 내역서 Excel 시트 구조 상세화

- `내역서 Excel 시트 구조` 항목도 기존보다 상세하게 작성한다.
- 각 시트의 목적, 포함 컬럼, 데이터 입력 방식, 자동 계산 여부, 참조 DB 여부를 설명한다.
- 구조만 나열하지 말고, 각 시트가 어떤 역할을 하는지 설명을 추가한다.

권장 구성:

| 시트명 | 목적 | 주요 컬럼 | 입력/자동화 방식 | 참조 DB | 비고 |
| --- | --- | --- | --- | --- | --- |
| 예: 내역서 | 최종 내역 수량 및 금액 정리 | 품목명, 규격, 단위, 수량, 단가, 금액 | 자동 생성/일부 수동 보정 | 품목 DB, 단가 DB | 필요 시 수정 |

### 4.4 삭제 대상 항목

다음 항목은 문서에서 삭제한다.

- `내역서 자동화 개발(IAAN)`
- `관리자 검토 워크플로`

## 최종 검수 기준

- 문서 구조가 기존보다 명확하게 정리되어 있어야 한다.
- 각 수정 요청은 실제 Notion 문서에 바로 반영할 수 있을 만큼 구체적이어야 한다.
- 삭제, 변경, 추가 항목이 서로 혼동되지 않아야 한다.
- 샘플 이미지 삽입 영역은 실제 이미지가 없어도 추후 추가 가능한 구조로 남겨야 한다.
- DB 연계, 산출 로직, Excel 시트 구조는 관계가 드러나도록 작성해야 한다.
- 전체 문서에서 오탈자, 어색한 표현, 의미가 불분명한 문장을 정리해야 한다.
