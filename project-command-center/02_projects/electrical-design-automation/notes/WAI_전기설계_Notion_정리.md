## 2. 전기설계 초기 데이터

### 초기 데이터 출처
전기설계 초기 데이터는 공정 설계 결과로 확정된 기계 리스트에서 나옵니다. 기계 리스트는 기계 카탈로그 DB를 기반으로 생성됩니다.

### 기계 카탈로그 전기 관련 컬럼
- 동력 (kW)
- 기동방식
- 정격전압 (V)
- 전동기 효율 (%)
- 수용율 (%)
- 극수 (P)

### 기동방식 코드
| 코드 | 기동방식 |
|------|----------|
| S_SYS01 | Direct-On-Line (DOL) 직입기동 |
| S_SYS02 | Star-Delta 기동 |
| S_SYS03 | Inverter(VFD) 기동 |
| S_SYS04 | Soft Starter 기동 |

### 데이터 흐름
공정 블록 구성 → 장비 선택 → 기계 리스트 확정 → 전기 설계 저장 → 동력설비 부하계산서 / 부하일람표 자동 생성

## 3. 전기 계산서 생성

공정 계획 탭에서 [전기 설계] 모드를 클릭하면 설정 팝업이 열립니다. 파라미터 설정 후 [저장]을 클릭하면 자동으로 산출물이 생성됩니다.

### 설정 팝업 파라미터

**수변전 설비 유무 선택:**
- **있음** → 저압반 + 고압반 전기설계 / 내역서에 수변전설비 시트(8-1, 8-2) 포함
- **없음** → 저압반 전기설계만 진행 / 수변전설비 시트 제외

**전기 조건 설정값:**
- 건축 부하: 기본값 20% (동력설비 부하 대비 건축 부하 적용율)
- MCC별 한계 상용 전류: 기본값 400A (MCC 그룹의 상한 전류값)
- 건축 설비 분전반 수용율: 기본값 100%

> MCC별 한계 상용 전류(400A)를 기준으로 장비를 자동 그룹핑합니다. 합산 전류가 한계를 초과하면 다음 MCC로 분리됩니다.

---

### ① 전기 계산서 구성

전기 계산서는 동력 계통 / 계측 계통 / IO 계통으로 구분됩니다.

| 계통 | 내용 |
|------|------|
| 동력 계통 | 초기 데이터 (기기번호/기기명/동력/수량) — 기계 카탈로그에서 자동 생성 |
| 계측 계통 | 계측기리스트 (계측기명/설치위치/수량/전선/PLC전선/외경) — P&ID에서 자동 생성 |
| IO 계통 | SBR IO MCC 및 계측제어 → IO 집계표 (DI/DO/AI/AO 합산 + 20% 예비 → PLC 카드 수량) |

| 기기명 | equipment_name | |

| 총 수량 | total_qty | normal_qty + spare_qty |

**카탈로그에서 자동으로 가져오는 항목:**
| 필드 | DB 컬럼 | 비고 |

- 측정대상물질
- 수량

### ③ 동력설비 부하계산서

**핵심 계산 공식 5종:**
| 계산 항목 | 공식 | 적용 기준 |
|-----------|------|-----------|
| 부하용량(kVA) | kW ÷ (η × cosφ) | KEC 210.5, KS C IEC 60034-1 |
| 정격전류(A) | kW × 1000 ÷ (√3 × 380 × η × cosφ) | KEC 230.4 |
| 상용전류(A) | 정격전류 × 상용수량 | MCC 단위 내 병렬 전류 합산 |
| MCC 대표 상용전류(A) | 구간 합계 + 최대 상용 전류 × 10% | 최대부하 기동 보상 |
| 수용동력(kW) | 상용동력 × 수용율 | |

**DB 구조:**
- 입력 (equipment_input): equipment_id, power_kw, normal_qty, spare_qty, efficiency, power_factor, demand_factor
- 출력 (load_calculation_result): equipment_id, apparent_kva, rated_current_a, normal_current_a, mcc_group_id

**조작반 구분 로직:**
- `s_ctrl_method = S_SYS04` → `panel_type = 'MOP'` (MCC로 관리하는 전동기 조작반)
- `s_ctrl_method ≠ S_SYS04` → `panel_type = 'LOP'` (Local Operation Panel — 현장 조작반)

### ④ 전선(CV/GV) 선정

**전선 종류:**
| 종류 | 용도 |
|------|------|
| F-CV | 동력 케이블 (3상 전동기 주전원) |
| F-CVV | 제어 케이블 (MCC 제어 배선) |
| F-CVV-SB | 차폐 제어/계측 케이블 (노이즈 방지) |
| F-GV | 접지 케이블 (보호도체) |

**선정 로직:**
- 입력: 상용전류(A) → cable_ampacity_rule 구간 매칭 → 허용전류 이상 최소 규격 선택
- 출력: cable_size_mm2, cable_cores
- 접지선(GV): 동력 케이블 규격 이하 표준 규격 선택 (KEC 140)

**추가 구현 필요 항목 (미적용):**
- 포설 방식 보정 (Cf): 공기 중 / 전선관 내 / 케이블 트레이 (KS C IEC 60364-5-52 Table B.52)
- 온도 보정 (Ct): 주위온도 30°C 초과 시 허용전류 감소
- 집합 계수 (Cg): 다회로 병설 시 상호 발열 보정
- 전압강하: 동력 회로 3% 이내 권장 (KEC 232.53)
- 단락전류 조건 (KEC 232)

### ⑤ 배전/제어기기 선정

**기동방식 코드별 제어기기 조합:**
| 기동방식 | 제어기기 구성 |
|----------|--------------|
| S_SYS01 (DOL 직입) | 차단기(MCCB) + 전자접촉기(MC) + 과부하계전기(EOCR) |
| S_SYS02 (Star-Delta) | 차단기 + MC×3개(주/와이/델타) + EOCR + 타이머 릴레이 |
| S_SYS03 (VFD 인버터) | 차단기 + 인버터 (EOCR 없음 — 인버터 내장 보호) |
| S_SYS04 (소프트스타터) | 차단기 + 소프트스타터 + EOCR |

**각 기기 선정 DB:**
| 기기 | DB 테이블 | 입력 | 주요 제조사 |
|------|-----------|------|------------|
| 차단기(MCCB) | breaker_selection_rule | 상용전류(A) | LS산전 (Susol, Metasol EBS) |
| 전자접촉기(MC) | magnetic_contactor_rule | 동력(kW) | LS산전 (GMC 시리즈) |
| EOCR | eocr_selection_rule | 동력(kW) + 기동방식 | LS산전 DMP 시리즈 |
| 인버터 | inverter_selection_rule | 동력(kW) | LS산전 (G100/S100) |
| 소프트스타터 | soft_starter_selection_rule | 동력(kW) | LS산전 (Styra) |
| 콘덴서 | capacitor_selection_rule | 동력(kW) + 타깃역률 | — |

### ⑥ MCC 선정

**MCC 패널 수량 산정:**
- 설정값: MCC별 한계 상용 전류 (UI 설정 파라미터로 변경 가능)
- 누적 상용전류 계산 → 설정값 초과 시 다음 MCC로 분리
- MCC 대표 상용전류 = 구간 합계 + 최대 상용 전류 × 10% (여유)
- 출력: mcc_group_id (예: MCC-1, MCC-2 ...), 각 장비의 mcc_group_id 할당

**MCC Unit 사이즈 선정:**
- 입력: 동력(kW) + 기동방식 코드 → mcc_unit_size_rule 매핑 → Unit 폭(mm) 선정
- 주요 필드: rule_id, power_kw_min, power_kw_max, start_method_code, unit_width_mm

### ⑦ 전선관 선정

**전선관 선정 4단계 흐름:**
1. 케이블 선정 완료 후 전선 규격 확정 (F-CV/F-CVV/F-CVV-SB)
2. 전선 규격 → cable_outer_diameter DB 룩업 → 외경(mm) 반환
3. 동일 전선관 내 케이블들의 단면적 합산 → 점유율 계산
4. conduit_fill_rule(점유율 기준) + conduit_selection_rule(규격) DB → 상위 규격 선정

**단면적 계산 방법:**
- 각 케이블 단면적(mm²) = π ÷ 4 × d²
- 점유율(%) = 전체 단면적 합계 ÷ 전선관 내면적 × 100

**표준 점유율 기준:**
| 전선 수 | 점유율 |
|---------|--------|
| 1가닥 | 53% 이하 |
| 2가닥 | 31% 이하 |
| 3가닥 이상 | 40% 이하 (사내 기준) |

**부속품 수량 산식:**
- 가요전선관 = ROUNDUP(강제전선관 수량 ÷ 6, 0)
- 박스커넥터 = ROUNDUP(강제전선관 수량 ÷ 6, 0)
- 노말밴드 = ROUNDUP(강제전선관 수량 ÷ 12, 0)

### ⑧ IO 집계

**IO 집계 흐름:**
- SBR IO MCC: 기동방식 코드별 DI/DO/AI/AO 포인트 수 → plc_signal_rule DB
- SBR IO 계측제어: 계측기 코드별 DI/DO/AI/AO 포인트 수 → instrument_signal_rule DB
- IO 집계표: 동력 IO + 계측 IO 유형별 DI/DO/AI/AO 합산
- 20% spare 적용 → 최종 IO 포인트 수 = 합계 × 1.2 (올림)
- PLC 카드 수량 = CEILING(spare 적용 IO ÷ 카드당 포인트 수)

**기동방식별 IO 포인트 (plc_signal_rule DB):**
| 기동방식 | DI | DO | AI | AO | 비고 |
|----------|----|----|----|----|------|
| S_SYS01 (DOL) | 2 | 1 | 0 | 0 | 운전/정지 피드백, 기동명령 |
| S_SYS02 (Star-Delta) | 2 | 2 | 0 | 0 | 와이/델타 전환 명령 추가 |
| S_SYS03 (VFD) | 2 | 1 | 1 | 1 | 속도 설정지령/피드백 |
| S_SYS04 (SS) | 2 | 1 | 0 | 0 | RS-485 Modbus 또는 하드와이어 IO |

**계측기 IO 포인트 (instrument_signal_rule DB):**
| 계측기 | AI | AO | DI | DO | 전원 케이블 |
|--------|----|----|----|----|------------|
| FT (유량계) | 1 | 0 | 0 | 0 | F-CVV-SB 1.5mm²×2C |
| LT (수위계) | 1 | 0 | 0 | 0 | 이중화 시 AI×2 |
| PT (압력계) | 1 | 0 | 0~1 | 0 | 상한하 알람 DI 추가 가능 |
| TT (온도계) | 1 | 0 | 0 | 0 | |
| FIC (유량제어계) | 1 | 1 | 0 | 0 | PID 제어 및 밸브 개도 피드백 |

## 4. 전기 도면 생성

전기 설계 설정 완료 후 [전기 도면] 모드가 활성화됩니다. (전기 설계 설정을 저장하지 않으면 비활성 상태 유지)

### 도면 생성 흐름
`[전기 도면]` 모드 진입 → 도면 항목별 `[SVG 생성]` 버튼 클릭 → SVG 편집기에서 도곽(Border) 설정 → `[Export as PDF/DXF]`

### 생성 가능 도면 종류

**공정 계획 탭 — 단일 SVG → 단일 PDF/DXF:**
- 전기 범례 및 일반 사항
- MCC 구성도
- 현장조작반 외형도
- 전동기 기동반 결선도

**3D 레이아웃 탭 — SVG 파일 수만큼 DXF 생성:**
- 전기실 배치도
- 케이블트레이 배치도

### 케이블 스케줄 3종
케이블 스케줄은 동력/제어/계측 3종이 있으며, 전기 계산서에 포함됩니다.
- 동력 케이블 스케줄
- 제어 케이블 스케줄
- 계측 케이블 스케줄

## 5. 3D 레이아웃 — 전기 트레이

### 3D 레이아웃 탭 모드 순서 (위→아래 순서로 작업)
1. 구조물/기계 편집 — 구조물 편집, 기계 배치
2. 적층/오픈홈 — 기초 콘크리트 생성, 토목 수량 산출
3. 배관 라우팅 — 배관 생성, 프리셋 불러오기
4. **전기 트레이** ← 전기 트레이 배치 및 전선 연결 (현재 파트)
5. 수리계통도 생성 — 배치 기반 수리계통도 생성
6. 도면 생성 — 배치 기반 도면 생성

### 전기 트레이 모드 세부 흐름

**1단계: MCC 배치**
- [MCC] 아이콘 클릭
- 배치 위치 선택 → MCC 생성

**2단계: Tray 배치**
- [TrayH] 클릭 → 수평 트레이 생성
- [TrayV] 클릭 → 수직 트레이 생성
- 객체 편집 메뉴로 회전 및 길이 설정

**3단계: 전선 연결**
- [커넥터] 아이콘 클릭 → 트레이 위 위치 선택
- C키 → 커넥터 생성
- 시작점·끝점 커넥터 선택 → [전선 생성] 클릭 → 전선 연결

> 전선 연결이 완료되면 트레이 및 케이블 경로 기준으로 길이값이 산출되며, 해당 길이값은 동력설비 부하계산서 작성 및 산출 로직과 연계됩니다.

### 도면 생성
전기 트레이 배치 완료 후 [도면 생성] 모드에서 SVG → PDF/DXF 내보내기 진행.

### 내역서 수량 산출 로직

전기 내역서는 전기 계산서 결과를 직접 복사하는 방식이 아닙니다. 동력설비 리스트, 계측기 리스트, 케이블 스케줄, 3D Tray 배치 결과를 기반으로 산출합니다.

#### 1. 전선
- 입력: 동력설비 리스트 JSON/API, 계측기 리스트 API, 전기 카탈로그 DB
- 산출 방식: 전선 규격 확인 → equipment_code 매칭 → 수량 집계 → 케이블 길이 적용 (3D 트레이 경로 길이 우선)

**산출 품목:**
- 0.6/1kV 난연 전력 케이블(F-CV)
- 0.6/1kV 트레이용 절연난연 제어용케이블(F-CVV)
- 0.6/3kV 트레이용 제어용케이블(F-CVV-SB)
- 난연 비닐 절연 접지용전선(F-GV)

#### 2. 전선관 및 부속품
- 입력: 동력설비 리스트 API (conduit_length_m, conduit_power, conduit_control), 계측기 리스트 API
- 부속품: 가요전선관 = ROUNDUP(강제전선관 / 6, 0), 박스커넥터 = ROUNDUP(강제전선관 / 6, 0), 노말밴드 = ROUNDUP(강제전선관 / 12, 0)

#### 3. 케이블트레이 및 부속품
- 입력: 3D 레이아웃에서 전달되는 Cable Tray equipment_code + Quantity
- 부속품: Elbow, Connector, 지지대, Hold down clamp, Cover 등

#### 4. 현장조작반 (LOP)
- 입력: 기동방식 (s_ctrl_method), LOP 수량
- 기동방식 매핑: S_SYS01→직입기동, S_SYS02→인버터, S_SYS03→소프트스타터, S_SYS04→MOP
- 내부 부속품: MCCB, PBL, SELECT S/W, EM S/W, 콘센트, NAME PLATE, PVC DUCT, KIV CAB

#### 5. 전동기제어반 (MCC)
- 입력: mcc_no, mcc_panel_count, 전기계산 결과
- 처리: mcc_no 중복 제거 → MCC별 panel_count 산출 → 본체/내부 부속품 수량 산출
- 내부 부속품: MCCB, Terminal Block, SPD, Condenser, Name Plate 등

#### 6. 계측기
- 입력: 계측기 equipment_type, equipment_code, 수량, 계측기 카탈로그 DB
- 대상: 수위계 / 유량계 / pH 측정기 / ORP 측정기 / DO 측정기 / 압력계 / 온도계 / MLSS계

### 기술 사항

**Notion API 핵심 패턴:**
- 토글 블록으로 테이블 감싸기 → 렌더링 지연(lazy rendering)으로 페이지 성능 개선
- 100행 초과 테이블: 테이블 생성 후 응답에서 table block ID 추출 → 나머지 행 100개씩 batch append
- 하이퍼링크 삽입: `{'type':'text','text':{'content':'라벨','link':{'url':'https://...'}}}`

**Excel 파싱 핵심 패턴:**
- 섹션 구분: ①②③ 유니코드 원문자(chr 0x2460~0x2466) 행 감지
- 데이터/비고 컬럼 분리: 헤더 행에서 첫 None 또는 '예시' 포함 셀 이후를 비고로 처리
- 비고가 전 행에서 비어 있으면 자동 제거

**참조표 파서 종류:**
- `parse_standard()` — 일반 룩업 테이블 (CV/GV, 차단기, MC, EOCR, 인버터, 콘덴서, 소프트스타터, MCC사이즈, 전선관)
- `parse_matrix()` — 2D 매트릭스 (전선외경 — 단면적 × 코어수)
- `parse_plc()` — 리프 노드 필터링 (PLC 전선 — is_leaf=True, 11컬럼 선택)
- `parse_plc_wire()` — 계측기 전선 사양 (전선 및 PLC 전선 — 15컬럼)
