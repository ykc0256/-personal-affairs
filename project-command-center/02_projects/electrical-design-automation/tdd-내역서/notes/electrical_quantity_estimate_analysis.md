## 내역서 양식의 전기 파트

`내역서(양식)_국문(Metric)_r11.xlsx` 중 전기설계 관련 시트는 다음이다.

| 시트 | 역할 |
| --- | --- |
| `6-1. 기자재내역서(전기)` | 전기 기자재 수량, 재료비 단가, 금액 산출 |
| `6-2. 설치비내역서(전기)` | 전기 설치비 수량, 노무비/경비 단가, 금액 산출 |
| `6-3. 일위대가목록(전기)` | 전기 품목별 호표, 품명, 규격, 단위, 재료비/노무비/경비 요약 |
| `6-4. 일위대가(전기)` | 호표별 세부 일위대가 구성 |

전기 내역서 양식의 기본 계산 구조:

```text
기자재내역서:
  Quantity × 재료비 Unit Price = 재료비 Total Price

설치비내역서:
  Quantity × 노무비 Unit Price = 노무비 Total Price
  Quantity × 경비 Unit Price = 경비 Total Price

합계:
  재료비 Total + 노무비 Total + 경비 Total
```

## 수량산출 항목별 생성 로직

### 1. 전선

참조 위치:

- `2-1. 전선`
- `3-1. 전선`

입력 데이터:

- 동력설비 리스트 JSON/API
- 계측기 리스트 API
- 전기 카탈로그 DB

주요 코드/값:

- `cable_cv`
- `cable_gv`
- `cable_cvv`
- `cable_cvv_sb`
- 전선 길이
- 전선 수량

산출 방식:

```text
전기계산서/케이블스케줄의 전선 규격
  -> 전기 카탈로그의 equipment_code와 매칭
  -> 같은 equipment_code별 수량 집계
  -> Quantity = 수량 × 길이
  -> 전선 품명/규격/단위/단가를 카탈로그에서 불러옴
```

예시 결과:

- `0.6/1kV 난연 전력 케이블(F-CV)`
- `0.6/1kV 트레이용 절연난연 제어용케이블(F-CVV)`
- `0.6/3kV 트레이용 제어용케이블(F-CVV-SB)`
- `난연 비닐 절연 접지용전선(F-GV)`

### 2. 전선관

참조 위치:

- `2-2. 전선관(신규)`
- `3-2. 전선관`

입력 데이터:

- 동력설비 리스트 API
  - `conduit_length_m`
  - `conduit_power`
  - `conduit_control`
- 계측기 리스트 API
  - `conduit_power`
  - `conduit_control`
- 전기 카탈로그 DB

중요 메모:

- 계측기 리스트 API에는 `conduit_length_m`이 없다고 명시되어 있다.
- 전선관 값은 `82C`, `22C`, `28C`처럼 숫자와 `C`가 조합된 형태다.

산출 방식:

```text
전선관 규격 값 확인
  -> 전기 카탈로그의 전선관 equipment_code 생성/매칭
  -> 강제 전선관 길이 산출
  -> 가요전선관, 박스커넥터, 노말밴드 등 부속품 수량 산출
```

예시 부속품 산식:

- 가요전선관: `ROUNDUP(강제전선관 길이 / 6, 0)`
- 박스커넥터: `ROUNDUP(강제전선관 길이 / 6, 0)`
- 노말밴드: `ROUNDUP(강제전선관 길이 / 12, 0)`

### 3. 케이블트레이

참조 위치:

- `2-3. 케이블트레이(신규)`
- `3-3. 케이블트레이`

입력 데이터:

- 3D에서 전달되는 Cable Tray equipment_code
- 3D에서 전달되는 수량
- 전기 카탈로그 DB

중요 메모:

- 케이블트레이는 3D에서 코드와 수량이 함께 넘어온다고 명시되어 있다.
- 수량산출서의 Tray 항목은 계산서보다 3D 배치 데이터에 더 직접적으로 의존한다.

산출 방식:

```text
3D Tray 배치
  -> Tray equipment_code + Quantity 전달
  -> 전기 카탈로그에서 품명/규격/단가 조회
  -> Tray 본품 수량 산출
  -> Elbow, Connector, 지지대, Hold down clamp, Cover 등 부속품 산출
```

### 4. 현장조작반

참조 위치:

- `2-4. 현장조작반`
- `3-4. 현장조작반`

입력 데이터:

- 기동방식 `s_ctrl_method`
- LOP 수량
- 전기 카탈로그 DB

기동방식 매핑:

- `S_SYS01`: 직입기동
- `S_SYS02`: 인버터
- `S_SYS03`: 소프트스타터
- `S_SYS04`: MOP

산출 방식:

```text
기동방식 확인
  -> LOP 필요 여부/회로수 판단
  -> LOP 본체 수량 산출
  -> MCCB, PBL, SELECT S/W, EM S/W, 콘센트, NAME PLATE, PVC DUCT, KIV CABLE, 압착단자 등 부속품 산출
```

### 5. 전동기제어반

참조 위치:

- `2-5. 전동기제어반(수정보완)`
- `3-5. 전동기제어반`

입력 데이터:

- `mcc_no`
- `mcc_panel_count`
- MCC 관련 전기계산 결과
- 전기 카탈로그 DB

중요 메모:

- `mcc_no` 값은 중복 제거해서 사용한다.
- 예: `MCC1`, `MCC2`가 여러 번 검색되더라도 내역서에는 한 번씩 반영한다.
- MCC 수량은 `mcc_panel_count` 값을 사용한다.

산출 방식:

```text
mcc_no 중복 제거
  -> MCC별 panel count 산출
  -> MCC 본체 항목 생성
  -> MCCB, Terminal Block, SPD, Condenser, Name Plate 등 부속품 산출
```

### 6. 계측기

참조 위치:

- `2-6. 계측기(수정보완)`
- 계측기 리스트 API
- 계측기 카탈로그 DB

입력 데이터:

- 계측기 `equipment_type`
- 계측기 `equipment_code`
- 계측기 수량

대상 예시:

- 수위계
- 유량계
- pH 측정기
- ORP 측정기
- DO 측정기
- 압력계
- 온도계
- MLSS계

산출 방식:

```text
계측기 리스트 API
  -> equipment_type 확인
  -> 계측기 카탈로그 DB에서 equipment_code, 품명, 규격, 단가 조회
  -> 계측기 수량 집계
  -> 전기 내역서 또는 계측 내역 항목으로 반영
```

## 설명자료에 반영할 문구

> 전기 내역서는 전기계산서의 계산 결과를 직접 복사하는 방식이 아니라, 동력설비 리스트, 계측기 리스트, 케이블 스케줄, 3D Tray 배치 결과를 수량산출서에서 품목별 수량으로 집계한 뒤, 전기/계측기 카탈로그 DB의 품명·규격·단가와 매칭해 내역서 양식의 전기 파트에 반영하는 구조다.

## 검토 포인트

- 전선 길이 기본값과 실제 3D 경로 길이의 우선순위를 정해야 한다.
- 계측기 리스트 API에 전선관 길이가 없는 경우 길이 산정 기준이 필요하다.
- Tray 수량은 3D에서 넘어오므로 3D 모델의 단위와 내역서 단위가 일치해야 한다.
- `equipment_code`가 모든 내역 항목의 핵심 키이므로 전기/계측기 카탈로그와 수량산출 결과의 코드 체계를 일치시켜야 한다.
- 전기 카탈로그의 `사용여부`가 False인 품목은 내역서 자동 산출에서 제외하거나 경고해야 한다.
- `단가출처`와 카탈로그 버전을 내역서 생성 이력에 남겨야 한다.
