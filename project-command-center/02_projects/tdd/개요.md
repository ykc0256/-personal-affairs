## 배경 및 목적

GWD 프로그램 개발이 막바지에 접어들었으나, 개발 종료가 끝이 아닌 시작점이다.

- 각종 피드백 수집 및 반영
- 공정 확장
- 오류 개선

이 후속 작업들을 원활하게 진행하기 위해, 각 파트별 책임자가 개발에 사용된 **아이디어, 룰, 표준자료**를 TDD로 정리한다. TDD는 프로젝트 시작 시 작성하는 문서이기도 하지만, 본 건은 **개발 종료 관점**에서 작성한다.

## TDD 목차 구성 (7개 섹션)

| TDD 섹션 | 최윤규 담당 내용 |
|---|---|
| 1. 개요 및 배경 | 전기설계 규칙 문서의 목적·범위·가정 |
| 2. 최종 시스템 아키텍처 | 이승호 주도, 전기설계 파트 데이터 흐름 기여 |
| 3. 공정규칙 및 계산 로직 | **전기설계 규칙** (계산 로직 전체) |
| 4. 파라메트릭 모델 및 3D 생성 규칙 | **2D 생성 규칙** (SVG 생성, 도면 연결규칙) |
| 5. 상세 설계 | **내역서 생성 규칙** (수량산출 로직, DB 연결) |
| 6. 구현 및 테스트 전략 | 소스코드 입수 후 보완 예정 |
| 7. 향후 고려사항 및 부록 | 작성 완료 후 추가 |
