## 0. 답변 기준 확정 사항

| 구분 | 확정 내용 |
|---|---|
| 전기설계 범위 | 전기계산서 산출과 도면 산출을 하나의 전기설계 TDD 안에서 다루되, 두 과정을 구분해서 작성한다. |
| 필수 도면 | MCC 구성도, 전동기기동반 결선도, 범례, 현장조작반 외형도 포함. |
| 제외 범위 | 내역서 산출 과정과 P&ID 2D 도면 산출 과정은 별도 TDD로 분리한다. |
| 핵심 관점 | 전기계산서와 도면 사이의 연결성이 가장 중요하다. |
| 입력 흐름 | 공정계획 단계에서 공정 선택 -> 기계 선택(`equipment_catalog`) -> `report.json`, `project.json` 생성 -> 전기계산서 및 도면 산출. |
| 수동 수정 우선순위 | 사용자가 수정한 값은 자동 refresh보다 우선하며 보존한다. |
| 0kW 설비 | 동력이 없는 설비는 전기계산서 산출 대상에서 제외한다. |
| 기준정보 우선순위 | DB 기준정보가 항상 우선이며, Excel은 참고자료로만 사용한다. |
| MCC 그룹핑 | 사람이 설정한 한계 상용 전류값을 기준으로 자동 분할한다. MCC 번호는 기기번호 순서에 따라 순차 자동 기입한다. |
| MCC 수동 수정 | 자동 그룹핑 이후 사용자가 MCC 그룹을 수정하는 기능은 없다. |
| 길이 산출 | 3D 경로가 있으면 실제 경로를 사용하고, 3D 경로가 없으면 0m로 지정한다. 단, MCC 또는 계측기는 30m 고정값을 사용한다. |
| 계측기 | 계측기 케이블/IO 집계는 전기설계에 포함한다. P&ID에서 생성된 계측기 tag를 입력으로 받는다. |
| 도면 파일 | SVG 형태로 생성되는 것으로 보이며, PDF/DXF 저장이 가능하다. 결과는 MinIO에 저장한다. |
| 계산서 최종 산출물 | `99_reference/electrical-design/originals/attachments/elec/1s1s1a1s_electricalcapacity.xlsx`를 최종 성과품 기준으로 본다. |

## 0-1. 추가 확인이 필요한 의문 사항

아래 항목은 답변 기준으로 방향은 잡혔지만, TDD 확정 전에 한 번 더 확인하면 좋다.

| # | 의문 사항 | 이유 |
|---|---|---|
| 1 | `범례` 도면의 데이터 원천 | 범례 도면이 어디에서 생성/저장되는지는 추가 확인 필요. 현재 OpenAPI에서 `mccd`, `mswd`, `lcpo`는 확인됐지만 범례 전용 API/DB 매핑은 아직 확인되지 않았다. |
| 2 | 도면 범위 정리 | MCC 구성도, 전동기기동반 결선도, 범례, 현장조작반 외형도는 전기설계 TDD 포함 도면으로 정리한다. 전기실 배치도/케이블트레이 배치도는 3D 환경 산출물로 별도 취급한다. |
| 3 | Tray 구간과 전선관 구간 분리 | 사용자 답변상 소스코드 확인 필요 항목이다. 내역서와도 연결되므로 우선 확인 대상이다. |
| 4 | 도면 번호/파일명 생성 규칙 | 샘플 자료 생성 후 추가 확인하기로 한 항목이다. |
| 5 | 기동방식 누락, 케이블/차단기 자동선정 실패 처리 | 0kW 제외 외 나머지 예외 처리는 소스코드 확인 필요. |
| 6 | 노션 전기설계 정리 자료의 정확한 경로 | MCC 한계 상용 전류 등 핵심 규칙의 원천으로 쓰려면 참조 경로를 TDD에 명시해야 한다. |

## 0-2. TDD 반영 방향

전기설계 TDD는 다음 구조로 정리한다.

1. 전기설계 범위
   - 전기계산서 산출
   - 전기 도면 산출
   - 두 산출물의 연결 구조

2. 입력 데이터 흐름
   - 공정 선택
   - 기계 선택
   - `equipment_catalog`
   - `report.json`
   - `project.json`
   - DB 전기 테이블

3. 전기계산서 산출
   - `1s1s1a1s_electricalcapacity.xlsx` 기준 산출물 구조
   - 0kW 설비 제외
   - 사용자 수정값 보존
   - DB 기준정보 우선

4. 도면 산출
   - MCC 구성도
   - 전동기기동반 결선도
   - 범례
   - 현장조작반 외형도
   - SVG 생성, PDF/DXF 저장, MinIO 저장

5. 소스코드 확인 필요 항목
   - Tray/전선관 구간 분리
   - 도면 템플릿 위치
   - 도면 번호/파일명 생성
   - 예외 처리
   - 범례 도면 생성 로직

## 1. 전기설계 범위

### Q1. 전기설계 TDD의 최종 산출 범위는 어디까지인가?

현재 추정:
- 전기계산서 산출 과정 포함
- 전기 도면 산출 과정 포함
- 내역서 산출 과정 제외
- P&ID 2D 도면 산출 과정 제외

확인 필요:
- 전기 도면 중 어디까지 전기설계 TDD에 포함할지 확정 필요

선택/답변 예시:
- 전기계산서 + MCC 구성도 + 결선도 + 현장조작반 외형도까지 포함
- 전기계산서만 우선 포함, 도면은 후속 단계
- 전기 평면도까지 포함

사용자 답변:

```
전기계산서 + MCC 구성도 + 전동기기동반결선도 + 범례 + 현장조작반 외형도 까지 포함하는 쪽으로 생각함. 
단, 전기계산서 산출 / 도면 산출 구분해서 하나의 TDD 파일에 녹여내는 것은 필수 
그리고 전기계산서와 도면 사이의 연결성이 매우 중요 

```

TDD 반영 위치:
- `1. 범위`
- `7. 전기 도면 산출 과정`

## 2. 입력 데이터 기준

### Q2. 전기설계의 시작 데이터는 무엇으로 볼 것인가?

현재 확인:
- API/DB 기준으로는 `project_equipment_selections`, `equipment_catalog`, `elec_equip_load_list`가 주요 시작점으로 보임
- 기존 자료에는 `report.json`, `project.json` 샘플도 존재함

확인 필요:
- 실제 프로그램에서 전기설계를 시작할 때 어느 데이터를 원천으로 삼을지 확정 필요

선택/답변 예시:
- 장비 선택 결과인 `project_equipment_selections`를 기준으로 시작
- 기존 `project.json`, `report.json` 구조도 유지
- 사용자가 전기설비 목록을 직접 입력하는 흐름도 허용

사용자 답변:

```
지금 과정을 개략적으로 설명해주겠음. 기본적으로 사용자메뉴얼을 확인하면 됨.

사람이 공정계획단계에서 공정을 선택함
그러면 그거에 따른 기계를 선택하게됨 (equipment_catalog)

이 데이터를 바탕으로 report.json 및 project.json이 생성됨

2 가지가 반영되서 최종적으로 전기계산서 및 도면이 완료됨

```

TDD 반영 위치:
- `3. 입력 데이터`
- `11. API/DB 기준으로 확정된 구현 연결`

## 3. 자동 계산값과 수동 수정값

### Q3. 자동 산출된 값을 사용자가 수정한 경우, refresh 시 어떻게 처리할 것인가?

현재 확인:
- API에는 자동채움/갱신 기능이 있음
- `refresh`, `batch_refresh`, `refresh-all` 계열 API 존재
- 다만 어떤 필드를 덮어쓰고 어떤 필드를 보존하는지는 API/DB만으로 확정 어려움

확인 필요:
- 사용자가 수정한 값을 refresh에서 유지할지, 자동값으로 다시 덮어쓸지 정책 필요

선택/답변 예시:
- 사용자가 수정한 값은 무조건 보존
- refresh 실행 시 자동 산출 필드는 모두 덮어쓰기
- 필드별로 보존/덮어쓰기 구분
- refresh 전에 사용자에게 덮어쓰기 여부 확인

사용자 답변:

```
사용자가 수정한 값은 무조건 보전 쪽이 맞음 (우선수위)

```

TDD 반영 위치:
- `6. 전기계산서 산출 과정`
- `12. 소스코드가 와야 확정되는 항목`

## 4. 부하계산 규칙

### Q4. 부하용량, 정격전류, 정상전류 계산식은 현재 샘플 계산서 기준을 그대로 따르는가?

- 정확한 반올림, 예외 처리, 전동기 외 부하 계산식은 소스코드 또는 샘플 계산서 검증 필요

확인 필요:
- 계산식 기준 문서를 무엇으로 볼지
- 반올림 자리수
- 0kW 또는 동력 미입력 장비 처리

사용자 답변:

```
샘플 계산서 참고해서 진행하면 될 것 같음.
또한 동력이 없는 설비의 경우, 전기계산서에서 제외되는 항목임

## 5. MCC 그룹핑

### Q5. MCC 그룹은 어떤 기준으로 나눌 것인가?

현재 추정:
- `mcc_no` 기준으로 그룹이 나뉨
- `field_type=TITLE` 행이 MCC 대표/집계 행 역할을 함
- 설비 행은 `field_type=ITEM`으로 보임
- MCC 기준 전류, 설비 순서, 위치 기준 분리 여부는 추가 확인 필요

확인 필요:
- MCC 분할 기준 전류
- 설비 정렬 순서
- 위치/구조물/공정 단위로 MCC를 분리할지
- MCC column 산정 방식

선택/답변 예시:
- 전류 기준으로 자동 분할
- 공정/위치/구조물 기준으로 우선 분리 후 전류 기준 적용
- 사용자가 MCC 번호를 직접 지정
- 자동 그룹핑 후 사용자가 수정 가능

사용자 답변:

```
MCC 분할 기준 전류는 사람이 설정하는 한계 상용 전류 값임. 이부분은 노션 전기설계 정리된 내용 참고 
한계 상용 전류값을 기준으로 분리 후 전류 기준 적용됨
MCC 번호의 경우, 기기번호 순서에 따라 순차적으로 MCC 번호가 자동으로 기입됨
자동 그룹핑 이후 사용자 수정하는 기능은 없음 

```

TDD 반영 위치:
- `6.3 MCC 그룹핑`
- `7. 전기 도면 산출 과정`

## 8. 계측기 전기설계 포함 범위

### Q8. 계측기 리스트는 전기설계 TDD에 어느 수준까지 포함할 것인가?

현재 확인:
- `elec_instrument_list` API/DB가 존재함
- 계측기 케이블, IO, 전선관 필드가 있음
- P&ID 도면 생성 자체는 별도 TDD 범위로 분리되어 있음

확인 필요:
- 계측기 전원/신호 케이블 산출을 전기설계에 포함할지
- IO 집계까지 전기설계에 포함할지
- P&ID에서 생성된 계측기 tag를 입력으로만 받을지

선택/답변 예시:
- 계측기 케이블/IO 산출까지 전기설계에 포함
- 계측기 tag는 P&ID에서 받고, 전기설계는 케이블만 산출
- 계측기는 후속 단계로 분리

사용자 답변:

```
계측기는 전기설계 포함됨
