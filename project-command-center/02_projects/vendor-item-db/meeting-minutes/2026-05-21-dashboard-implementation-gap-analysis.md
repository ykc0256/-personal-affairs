## 현재 구현 요약

현재 대시보드는 로그인과 주요 조회 화면 중심의 1차 버전이다. DB 정의서의 핵심 테이블은 Prisma 모델로 연결되어 있고, 화면 설계서의 16개 화면 중 6개 화면이 실제 라우트로 구현되어 있다.

구현된 라우트는 `/`, `/login`, `/equipments`, `/vendors`, `/prices`, `/evaluations`, `/admin`이다. `/admin`은 관리 메뉴 허브 역할만 하며, 실제 사용자 관리, 업로드, 분류 관리 화면은 아직 없다.

| 영역 | 구현 내용 |

## 주요 차이점 및 리스크

| 항목 | 현재 상태 | 영향 |
|---|---|---|
| 상세 화면 부재 | 기자재/업체/평가 상세 라우트 없음 | 목록에서 드릴다운 업무 불가 |
| 등록/수정 기능 부재 | 업체, 기자재, 단가, 평가, 사용자 입력 화면 없음 | 실제 운영 데이터 관리 불가 |
| Excel 업로드 미구현 | 링크만 있고 화면/처리 로직 없음 | 대량 데이터 반영 불가 |
| 업로드 이력 미구현 | `excel_uploads` 카운트 외 화면 없음 | 오류 추적 및 재처리 불가 |
| 파일 첨부 미구현 | `attachments` 모델만 존재 | 카탈로그/견적서/인증서 관리 불가 |
| DB role 권한 미검증 | 앱 레벨 role 제어는 있으나 DB role GRANT 검증은 별도 필요 | 직접 DB 접근 방어 확인 필요 |
| 한글 인코딩 깨짐 | 일부 TSX/문서 표시 문자열이 깨진 상태 | 화면 품질과 유지보수성 저하 |
| 필터 부족 | 설계서의 유형/국가/단위/등급/기간 필터 대부분 미구현 | 실무 검색 효율 낮음 |

## 다음 구현 우선순위

1. 한글 표시 문자열 인코딩 정리
   - 현재 화면 제목, 버튼, 설명 문구가 깨져 보이는 파일이 있다.
   - 우선 `src/app/(main)`, `src/components/layout/gnb.tsx`, `src/auth.ts`의 UI 문자열을 정상 한글로 복구한다.

2. 상세 화면 구현
   - `/equipments/[id]`: 기자재 기본 정보, 취급 업체, 최신 설계가/실행가, 평가 우선순위 표시
   - `/vendors/[id]`: 업체 기본 정보, 취급 기자재, 가격 이력, 평가 이력, 첨부 파일 표시

3. 입력/수정 업무 구현
   - 업체 등록/수정, 기자재 등록/수정, 단가 신규 등록
   - 설계서 기준상 단가는 수정이 아니라 신규 이력 추가 방식으로 처리한다.

4. 평가 입력 기능 구현
   - 평가 항목을 `evaluation_criteria`에서 로드
   - 항목별 점수 입력, 총점 자동 계산, 등급 자동 산정

5. Excel 업로드 및 이력 구현
   - 업체 등록 파일과 기자재/단가 등록 파일을 분리
   - 업로드 성공/실패 건수와 실패 row를 `excel_uploads`, `upload_error_rows`에 기록

6. 사용자/분류 관리 구현
   - `/admin/users`: 계정 생성, role 변경, 활성/비활성, 비밀번호 재설정
   - `/admin/categories`: 4단계 기자재 분류 트리 관리

7. 권한 검증 강화
   - 앱 레벨 role 제어와 PostgreSQL role 권한을 함께 점검
   - 특히 `execution_prices`는 engineer/viewer가 조회할 수 없어야 한다.
