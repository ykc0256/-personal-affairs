## 1. 방향 결정

Vendor Item DB는 단순 조회형 DB보다 **업무 판단용 DB**로 설계한다.

단순히 업체와 기자재를 검색하는 수준에 머무르면 사용자가 다음 행동을 판단하기 어렵다. 따라서 화면과 DB는 아래 질문에 답할 수 있어야 한다.

| 업무 질문 | 중심 화면 | 연결 데이터 |
|---|---|---|
| 이 기자재/모델은 어떤 업체가 공급하고, 단가는 어떻게 변했는가? | 기자재 | `equipments`, `vendor_items`, `design_prices`, `execution_prices` |
| 이 업체는 신뢰할 수 있는가? 최근 평가는 어떤가? | 업체 | `vendors`, `vendor_evaluations`, `evaluation_scores` |
| 지금 관리자가 처리해야 할 데이터 문제는 무엇인가? | 대시보드 | 미등록 단가, 평가 누락, 업로드 오류, 오래된 평가 |

---

## 2. 메뉴 구조 원칙

상단 메뉴는 기능 단위가 아니라 업무 판단 단위로 둔다.

```text
대시보드
기자재
업체
관리
```

| 기존 메뉴 | 변경 방향 | 이유 |
|---|---|---|
| 단가 비교 | 1차 메뉴에서 제외 | 단가는 기자재를 선택한 뒤 보는 정보가 자연스러움 |
| 업체 평가 | 1차 메뉴에서 제외 | 평가는 업체를 선택한 뒤 보는 정보가 자연스러움 |
| 관리 | 유지 | 입력/수정/업로드/사용자/분류 관리 기능의 진입점 |

`단가 비교`는 향후 가격 추세 분석, 업체별 랭킹, 엑셀 다운로드 등 분석 업무가 확정되면 `가격 분석` 메뉴로 재검토한다.

`업체 평가 목록`은 향후 평가 감사, 기간별 평가 리포트, 평가 누락 관리 수요가 커지면 `평가 리포트`로 재검토한다.

---

## 3. 화면별 역할

### 기자재

기자재 화면의 핵심은 **모델/규격별 가격 판단**이다.

- 좌측 기계 분류는 라이브러리형 접기/펼치기 트리로 구성한다.
- 기자재 목록에서는 취급업체수와 단위를 제거한다.
- 목록에는 모델/기자재명, 규격·사양, 대표 업체, 최고평가를 표시한다.
- 기자재명 클릭 시 행 아래 회색 확장 패널을 열고 단가 변경 이력을 보여준다.
- admin/procurement는 이 패널에서 단가를 추가하거나 오입력을 수정한다.

### 업체

업체 화면의 핵심은 **업체 신뢰도 판단과 보유 기자재 확인**이다.

- 업체 화면도 기자재 화면과 같은 분류 트리를 사용한다.
- 선택한 분류의 기자재를 취급하는 업체만 목록에 표시한다.
- 업체 목록에서는 설계가/실행가를 표시하지 않는다.
- 업체명 클릭 시 행 아래 회색 확장 패널을 열고 기본 정보, 평가 이력, 보유 기자재를 보여준다.
- admin/procurement는 이 패널에서 업체 평가를 등록/수정한다.

### 대시보드

대시보드는 단순 카운트보다 **처리해야 할 문제를 드러내는 화면**으로 둔다.

우선 표시할 지표:

| 지표 | 목적 |
|---|---|
| 평가 미등록 업체 수 | 평가 관리 누락 발견 |
| 평가 오래된 업체 수 | 재평가 대상 발견 |
| 단가 미등록 기자재 수 | 가격 판단 불가 항목 발견 |
| 최근 단가 변경 건수 | 가격 변동 감지 |
| 업로드 실패 건수 | 데이터 적재 오류 처리 |
| 출처 없는 단가 수 | 견적서/근거 파일 누락 발견 |

## 4. DB 보완 필요 사항

| 구분 | 보완 사항 | 이유 |
|---|---|---|
| 기자재 | `model_name` 또는 `manufacturer_model_no` 검토 | 기자재명과 실제 제조사 모델/형번이 다를 수 있음 |
| 단가 | 수정/무효화 이력 컬럼 검토 | 오입력 정정, 감사 추적 필요 |
| 단가 | 첨부파일과 단가 직접 연결 강화 | 가격 근거 견적서 추적 필요 |
| 평가 | 평가 점수 수정 이력 검토 | 평가 데이터는 의사결정에 영향 |
| 대시보드 | 미등록/오류/오래된 데이터 식별용 뷰 검토 | 업무 판단용 집계 성능과 일관성 확보 |

---

## 5. 우선순위

1. 화면설계서 기준으로 `기자재`, `업체`, `대시보드`, `관리` 구조를 확정한다.
2. DB 정의서에 모델명/형번, 단가 정정, 평가 정정, 첨부 연결 보완안을 반영한다.
3. 기존 구현된 `/prices`, `/evaluations`는 1차 메뉴에서 제외하고, 기능은 기자재/업체 확장 패널로 흡수한다.
4. 대시보드는 처리 필요 항목 중심으로 재설계한다.
5. 이후 관리 CRUD와 Excel 업로드를 구현한다.
