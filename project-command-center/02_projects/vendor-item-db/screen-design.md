## SCR-01 로그인

**접근:** 비로그인 상태. 로그인 후 `/` 로 redirect.

```
┌─────────────────────────────────────┐
│                                     │
│          [서비스 로고 / 타이틀]      │
│                                     │
│   아이디  ┌───────────────────────┐ │
│           └───────────────────────┘ │
│   비밀번호 ┌──────────────────────┐ │
│           └───────────────────────┘ │
│                                     │
│          [  로그인  ]               │
│                                     │
│   ⚠ 오류 메시지 영역               │
│                                     │
└─────────────────────────────────────┘
```

| 요소 | 타입 | 설명 |
|---|---|---|
| 아이디 입력 | text input | `users.username` |
| 비밀번호 입력 | password input | `users.password_hash` 검증 |
| 로그인 버튼 | button | 세션 발급, `/` redirect |
| 오류 메시지 | text | 아이디 없음 / 비밀번호 불일치 / 비활성 계정 |

> **[코멘트]**
> 회원가입 기능 없음. admin이 SCR-15에서 직접 계정 생성.

## SCR-02 대시보드

**접근:** 전체 역할

```
┌─────────────────────────────────────────────────────────────┐
│  평가 미등록   단가 미등록   업로드 실패   최근 단가 변경    │  ← 처리 카드
│  [  12개   ]  [   34개   ]  [   3건    ]  [   8건       ]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [처리 필요 업체]                 [처리 필요 기자재]         │
│  ┌──────────────────────────────┐  ┌──────────────────────┐ │
│  │ 업체명       사유      날짜  │  │ 기자재명  사유  날짜 │ │
│  │ 동화엔지     평가오래됨 05-18│  │ 원심펌프  단가없음 - │ │
│  │ 한국펌프     평가없음  -     │  │ 볼류트    출처없음 - │ │
│  │ ...          ...       ...   │  │ ...      ..  ...      │ │
│  └──────────────────────────────┘  └──────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 요소 | 연결 테이블 | 설명 |
|---|---|---|
| 처리 카드 | vendors / equipments / excel_uploads / vendor_evaluations / prices | 관리자가 조치해야 할 항목 집계 |
| 평가 미등록 업체 | vendors LEFT JOIN vendor_evaluations | 평가 이력이 없는 활성 업체 수 |
| 평가 오래된 업체 | vendor_evaluations | 마지막 평가일이 기준 기간보다 오래된 업체 |
| 단가 미등록 기자재 | equipments LEFT JOIN design_prices | 설계가가 없는 활성 기자재 |
| 업로드 실패 | excel_uploads / upload_error_rows | 실패 또는 부분 성공 업로드 |
| 출처 없는 단가 | design_prices / execution_prices | source 또는 첨부파일 연결이 없는 단가 |

> **[코멘트]**
> 대시보드는 단순 카운트보다 업무 판단용 화면으로 둔다. 핵심은 "지금 무엇을 처리해야 하는가"를 보여주는 것이다.

## SCR-03 기자재 조회

**접근:** 전체 역할

```
┌──────────────┬──────────────────────────────────────────────────────┐
│  [분류 트리] │  🔍 [모델명 / 기자재명 / 규격·사양 / 업체명 검색] [검색]│
│  라이브러리  │                                                      │
│              │  [업체유형▼] [국가▼] [활성만 보기 ☑]                │
│  ▼ 기계      │  ────────────────────────────────────────────────    │
│    ▼ 펌프류  │  모델/기자재명     규격·사양          대표 업체  최고평가 │
│      ▶ 수중펌프  ┌──────────────────────────────────────────────┐  │
│      ▶ 부스터펌프│ 수중오수펌프  DN80 / 양정20m / 5.5kW 동화엔지 ★A  │ │
│    ▶ 포기기류│  │   └ 선택 시 아래에 회색 상세 패널 확장          │ │
│    ▶ 교반기류│  │ 수중오수펌프  DN100/ 양정30m / 7.5kW 한국펌프 ★B │ │
│  ▶ 배관      │  │ 터보송풍기   Q=50㎥/min / 15kW       -        -  │ │
│  ▶ 전기      │  └──────────────────────────────────────────────┘   │
│              │                              [< 1 2 3 ... >]        │
└──────────────┴──────────────────────────────────────────────────────┘
```

> 같은 기자재명이라도 규격·사양이 다르면 별도 행으로 표시한다. 목록의 "모델/기자재명"은 현재 DB 기준 `equipments.equipment_name`을 표시한다. 실제 제조사 모델명/형번을 별도 관리해야 하면 `equipments`에 `model_name` 또는 `manufacturer_model_no` 필드를 추가 검토한다.

| 요소 | 연결 | 설명 |
|---|---|---|
| 분류 트리 | equipment_categories (depth 1→2→3→4) | 라이브러리처럼 접기/펼치기. 선택한 분류와 하위 분류만 필터 |
| 트리 기본 상태 | equipment_categories.depth | 최초 진입 시 depth 1은 펼침. 선택된 분류의 상위 경로는 자동 펼침 |
| 트리 노드 동작 | category_id | 화살표 클릭은 접기/펼치기, 분류명 클릭은 해당 분류 선택/필터 적용 |
| 선택 상태 유지 | URL category query | 기자재에서 `전체 기자재 > 기계 > 펌프 > 수중펌프` 선택 후 업체로 이동하면 업체 화면에서도 같은 분류가 선택된 상태로 유지 |
| 선택 표시 | selected category_id | 별도 `선택` 버튼을 두지 않고, 선택된 분류 행의 배경/글자색을 바꿔 가시적으로 표시 |
| 하위 분류 표시 | equipment_categories.parent_category_id | 현재 DB에 하위 항목이 없으면 기계/배관/전기 최상위 폴더만 표시. 하위 분류가 등록되면 같은 트리 안에 자동 노출 |
| 검색 | equipment_name + **specification** + vendor_name + vendor_code | 규격·사양과 업체명 포함 검색 |
| 필터 드롭다운 | vendor_type / country | 업체유형, 국가 중심. 단위 필터는 제외 |
| 활성만 보기 | equipments.is_active | 기본값 ON |
| 규격·사양 열 | equipments.specification | 목록에서 바로 확인 (핵심) |
| 대표 업체 | vendor_items + vendor_evaluations | 최고 평가 업체를 우선 표시. 평가가 없으면 최신 설계가가 있는 업체를 표시 |
| 설계가 | design_prices | 대표 업체 기준 최신 설계가 표시 |
| 실행가 | execution_prices | 대표 업체 기준 최신 실행가 표시 |
| 최고평가 | vendor_evaluations MAX(grade) | 해당 기자재 취급 업체 중 최고 등급. 왼쪽에 해당 업체명을 함께 표시 |
| 행 선택 | equipments.equipment_id | 페이지 이동 대신 선택 행 아래에 회색 상세 패널 확장 |

> **[코멘트]**
> 기자재 목록에서 취급업체수와 단위는 제거한다. 사용자가 궁금한 것은 "이 모델/규격이 어떤 업체 제품이고 평가가 어떤지"이므로 대표 업체와 최고평가를 붙여 보여준다.

## SCR-04 기자재 행 확장 상세

**접근:** 전체 역할 (실행가 열은 admin/procurement만 표시)

```
목록 행 선택 전
┌─────────────────────────────────────────────────────────────┐
│ 모델/기자재명     규격·사양               대표 업체  최고평가 │
│ 수중오수펌프      DN80 / 양정20m / 5.5kW  동화엔지   ★A      │
│ 수중오수펌프      DN100 / 양정30m / 7.5kW 한국펌프   ★B      │
└─────────────────────────────────────────────────────────────┘

목록 행 선택 후
┌─────────────────────────────────────────────────────────────┐
│ 모델/기자재명     규격·사양               대표 업체  최고평가 │
│ 수중오수펌프      DN80 / 양정20m / 5.5kW  동화엔지   ★A      │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 회색 상세 패널                                   [닫기] │ │
│ │ 코드: M_PMP0301      GWD ID: GWD-001                    │ │
│ │ 규격: DN80 / 양정20m / 5.5kW / 재질 SUS                 │ │
│ │                                                         │ │
│ │ 단가 변경 이력                                           [단가 관리]* │ │
│ │ 기준일      업체명          설계가      실행가      설계가변동 실행가변동 출처 상태 │ │
│ │ 2026-04-01 동화엔지니어링  3,200,000  2,800,000  +100,000  +50,000  견적서 최신 │ │
│ │ 2025-12-01 동화엔지니어링  3,100,000  2,750,000  -         -        견적서 이전 │ │
│ │ 2026-03-15 한국펌프공업    3,450,000  3,100,000  +50,000   -        견적서 최신 │ │
│ │ 2026-02-20 대성기계        3,500,000  -          -         -        견적서 정정제외 │ │
│ │                                                         │ │
│ │ [단가 추가]* [선택 단가 수정]* [첨부파일]                 │ │
│ └─────────────────────────────────────────────────────────┘ │
│ 수중오수펌프      DN100 / 양정30m / 7.5kW 한국펌프   ★B      │
└─────────────────────────────────────────────────────────────┘
* admin/procurement만 표시
** 실행가 열: admin / procurement만 표시
```

> 별도 상세 페이지(`/equipments/[id]`)는 유지할 수 있지만, 기본 조회 UX는 행 확장 방식으로 둔다. 사용자는 화면 전환 없이 선택한 제품 아래에서 상세 정보를 확인하고, 아래 행은 자연스럽게 밀려 내려간다.
> 기자재명 클릭 시 가장 중요한 상세 정보는 단가 변경 이력이다. 이력은 `구분(설계가/실행가)`을 첫 열로 나누지 않고, `기준일 + 업체`를 한 행의 기준으로 둔다. 같은 기준일·업체 조합 안에서 설계가와 실행가를 나란히 보여주고, 각각의 전회 대비 변동을 표시한다.
> 관리자는 같은 패널 안의 단가 관리 영역에서 단가를 추가하거나 오입력 정정을 처리한다. 단가 관리는 단가 비교 화면으로 이동하지 않고, 현재 선택한 기자재의 기준일·업체 단가 이력에 직접 반영되는 구조로 둔다.

**우선순위 정렬 로직:**

| 우선순위 | 기준 | 비고 |
|---|---|---|
| 1순위 | 평가 총점 높은 순 | vendor_evaluations.total_score |
| 2순위 | 최신 설계가 낮은 순 | 동점 시 |
| 3순위 | 평가 없는 업체 | 하단 배치 |
| 비활성 업체 | 최하단 | is_active = false |

정렬 드롭다운 옵션: `평가점수 높은순` / `설계가 낮은순` / `등록일순`

**단가 변경 이력 표시 기준:**

| 표시 열 | 설명 |
|---|---|
| 기준일 | 단가 기준일. 이력의 1차 정렬 기준 |
| 업체명 | 해당 단가를 제공한 업체 |
| 설계가 | 해당 기준일·업체의 설계가. 없으면 `-` |
| 실행가 | 해당 기준일·업체의 실행가. 없으면 `-` |
| 설계가 변동 | 같은 업체의 직전 설계가 대비 증감 |
| 실행가 변동 | 같은 업체의 직전 실행가 대비 증감 |
| 출처 | 견적서, 업로드 파일, 수기 입력 등 |
| 상태 | 최신 / 이전 / 정정 제외 |
| 비고 | 통화, 정정 사유, 입력 메모 등 |

**관리자 단가 반영 정책:**

| 작업 | 권한 | DB 반영 |
|---|---|---|
| 단가 추가 | admin / procurement | `design_prices` 또는 `execution_prices` 신규 행 생성 |
| 단가 관리 | admin / procurement | 기자재 확장 패널 내부에서 업체, 기준일, 설계가, 실행가, 출처, 비고를 입력 |
| 단가 수정 | admin / procurement | 정상 이력은 덮어쓰지 않고 신규 이력 추가. 오입력은 정정 제외 처리와 정정 사유 기록 |
| 단가 삭제 | admin | 실삭제보다 비활성/정정 이력 방식 우선 검토 |

> **[코멘트]**
> 모델명/기자재명을 클릭하면 행 확장 패널을 연다. 같은 행을 다시 클릭하거나 상세 패널의 `닫기` 버튼을 누르면 접는다. 다른 행을 클릭하면 기존 상세는 접고 새 상세를 연다. 다중 펼침은 목록이 길어질 수 있으므로 기본은 단일 펼침으로 한다.

## SCR-05 업체 조회

**접근:** 전체 역할

```
┌──────────────┬──────────────────────────────────────────────────────┐

│  라이브러리  │                                                      │
│              │  [업체유형▼] [국가▼] [재무등급▼] [활성만 보기 ☑]    │
│  ▼ 기계      │  ────────────────────────────────────────────────    │
│    ▼ 펌프류  │  업체명           코드        유형    국가  최근평가  │
│      ▶ 수중펌프  ┌──────────────────────────────────────────────┐  │
│      ▶ 부스터펌프│ 동화엔지니어링  VM_DH_ENG  물품   국산   A     │ │

│  ▶ 배관      │  │ 대성기계        VM_DS_MCH  물품   국산   B     │ │
│  ▶ 전기      │  └──────────────────────────────────────────────┘   │
│              │                              [< 1 2 3 ... >]        │
```

| 요소 | 연결 | 설명 |
|---|---|---|
| 분류 트리 | equipment_categories (depth 1→2→3→4) | 기자재 조회와 동일한 접기/펼치기 구조. 선택한 분류의 기자재를 취급하는 업체만 필터 |
| 전체 라벨 | equipment_categories 전체 | 업체 자체 분류가 아니라 기자재 기준 필터이므로 `전체 업체`가 아니라 `전체 기자재`로 표시 |
| 트리 기본 상태 | equipment_categories.depth | 최초 진입 시 depth 1은 펼침. 선택된 분류의 상위 경로는 자동 펼침 |
| 선택 상태 유지 | URL category query | 기자재 화면에서 선택한 기자재 분류를 업체 화면에서도 그대로 유지. 업체에서 기자재로 돌아갈 때도 동일 |
| 선택 표시 | selected category_id | 별도 `선택` 버튼 없이 분류명 클릭으로 선택하고, 선택된 분류 행의 배경/글자색을 바꿔 표시 |
| 검색 | vendors.vendor_name / vendor_code + equipments.equipment_name + equipments.specification | 업체명/코드와 취급 기자재명/규격 검색 |
| 업체유형 필터 | vendors.vendor_type | 물품/공사/용역/기타 |
| 국가 필터 | vendors.country | |
| 재무등급 필터 | vendors.financial_grade | |
| 최근평가 | vendor_evaluations 최신 건 | 최근 평가 등급만 목록에서 요약 표시 |
| 업체명 선택 | vendors.vendor_id | 페이지 이동 대신 선택 행 아래에 회색 상세 패널 확장 |

> 업체 목록에서는 설계가와 실행가를 표시하지 않는다. 업체가 보유한 기자재와 가격 조합이 많기 때문에 목록에서 단가를 대표값처럼 보여주면 오해가 생길 수 있다. 가격은 기자재 행 확장 패널의 단가 변경 이력에서 확인한다.

> **[코멘트]**
> 업체 목록도 기자재 조회와 같은 좌측 분류 라이브러리를 사용한다. 예를 들어 "기계 > 펌프류 > 수중펌프"를 선택하면 해당 분류의 기자재를 취급하는 업체만 보인다.

## SCR-07 단가 비교

**상태:** 보류. 1차 상단 메뉴에서 제외.
**변경 방향:** 독립 단가 비교 화면보다 기자재 행 확장 상세의 `단가 변경 이력`으로 흡수한다.

```
기자재 조회 → 기자재명 클릭 → 회색 확장 패널
└─ 업체별 최신 단가
└─ 단가 변경 이력
└─ admin/procurement 단가 추가·수정
```

| 요소 | 연결 | 설명 |
|---|---|---|
| 최신 단가 | design_prices / execution_prices | 별도 영역을 만들지 않고 단가 변경 이력 내 `최신` 상태로 표시 |
| 단가 변경 이력 | design_prices / execution_prices | 기준일 + 업체를 한 행의 기준으로 두고 설계가, 실행가, 각 변동액, 출처, 상태, 비고를 표시 |
| 단가 관리 | design_prices / execution_prices | 단가 비교 화면으로 이동하지 않고 기자재 확장 패널 안에서 신규 단가 추가 또는 오입력 정정 |

> **[코멘트]**
> 단가 비교가 필요한 이유가 명확해지기 전까지 별도 화면으로 두지 않는다. 나중에 전 기간 가격 추세, 업체별 랭킹, 엑셀 다운로드 같은 분석 업무가 확정되면 `가격 분석` 화면으로 재검토한다.

## SCR-09 업체 평가 입력 / 상세

**접근:** admin / procurement
**상태:** 별도 페이지 대신 업체 행 확장 패널 내부 관리 기능으로 설계.

## SCR-11 기자재 등록 · 수정

## SCR-13 Excel 업로드

**접근:** admin / procurement
**현재 구현:** 실제 업로드 화면은 아직 미구현. `/admin` 화면에는 준비 중 카드와 예정 경로만 표시해 미구현 라우트로 이동하지 않도록 처리.

> **업로드 파일 2종으로 분리 확정**
> - **업체 등록 파일**: 업체 정보 + 업체 코드 등록
> - **기자재 등록 파일**: 기자재 정보 + 취급 업체 코드 연결 + 설계가 + 실행가

---

### 파일 A — 업체 등록 템플릿 (시트 1개)

| 시트명 | 포함 데이터 | 주요 컬럼 |
|---|---|---|
| 업체 | vendors | vendor_code ★, vendor_name, vendor_type, country, business_no, financial_grade, revenue, capital_size, revenue_base_year |

> `vendor_code`는 업체 식별 기준 — 중복 시 업데이트, 신규 시 INSERT

---

### 파일 B — 기자재 등록 템플릿 (시트 2개)

| 시트명 | 포함 데이터 | 주요 컬럼 |
|---|---|---|
| 기자재 | equipments | equipment_code ★, equipment_name, category_code, unit, specification |
| 단가 | vendor_items + design/execution_prices | equipment_code ★, **vendor_code** ★, design_price, execution_price, price_date, source |

> 단가 시트의 `vendor_code` 기재 → 해당 업체-기자재 연결(vendor_items) 자동 생성  
> 업로드 처리 순서: 기자재 → 단가 (vendor_code는 사전 등록된 것만 허용)

---

```
┌─────────────────────────────────────────────────────────────┐
│  Excel 업로드                                               │
│                                                             │
│  업로드 유형 선택                                           │
│  ┌─────────────────────────┐  ┌─────────────────────────┐  │
│  │  업체 등록              │  │  기자재 등록             │  │
│  │  (시트 1개)             │  │  (기자재 + 단가 시트 2개)│  │
│  │  [템플릿 다운로드 ↓]    │  │  [템플릿 다운로드 ↓]    │  │
│  └─────────────────────────┘  └─────────────────────────┘  │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │   파일을 여기에 드래그하거나 클릭하여 선택하세요     │  │
│  │              (.xlsx, .xls 지원)                      │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  [파일 선택 후 시트별 미리보기]                             │
│  [탭: 기자재(12행)] [탭: 단가(24행)]                        │
│                                                             │
│  equipment_code │ vendor_code │ design_price │ exec_price  │
│  M_PMP0301      │ VM_DH_ENG   │  3,200,000   │  2,800,000  │
│  M_PMP0301      │ VM_KP_IND   │  3,450,000   │          -  │
│                                                             │
│  ⚠ 단가 시트의 vendor_code는 사전 등록된 업체만 허용       │
│                                                             │
│                                [  업로드 실행  ]           │
└─────────────────────────────────────────────────────────────┘
```

> **[코멘트]**
> 업로드 2종 분리 확정: 업체 등록(1파일) / 기자재+단가 등록(1파일). vendor_code가 두 파일의 연결 키.

## SCR-16 기자재 분류 관리

**접근:** admin
**현재 구현:** 실제 분류 관리 화면은 아직 미구현. `/admin` 화면에는 준비 중 카드와 예정 경로만 표시.

**4단계 계층 구조:**

| depth | 의미 | 예시 | 코드 예시 |
|---|---|---|---|
| 1 | 최상위 (분야) | 기계 / 배관 / 전기 | DISC_M / DISC_P / DISC_E |
| 2 | 대분류 | 펌프류 / 포기기·송풍기류 | M_PUMP / M_AEBL |
| 3 | 중분류 | 수중펌프 / 부스터펌프 | M_PMP03 / M_PMP07 |
| 4 | 소분류 | 수중오수펌프 / 수중오수펌프(SUS) | M_PMP0301 / M_PMP0302 |

```
┌──────────────────────────────┬──────────────────────────────────────┐
│  분류 트리                    │  분류 편집                           │
│                              │                                      │
│  ▼ 기계  (depth 1) [+ 추가] │  분류명*  [수중오수펌프            ] │
│    ▼ 펌프류        [+ 추가]  │  코드*    [M_PMP0301              ] │
│      ▼ 수중펌프    [+ 추가]  │  상위분류 [수중펌프 (M_PMP03) ▼   ] │
│        · 수중오수펌프[✎][🗑] │  깊이     4 (소분류) — 자동        │
│        · 수중오수(SUS)[✎][🗑]│  정렬순서 [  1  ]                  │
│      ▶ 부스터펌프  [+ 추가]  │  활성여부 [● 활성 ○ 비활성]        │
│    ▶ 포기기·송풍기류[+ 추가] │                                      │
│    ▶ 교반기류      [+ 추가]  │           [저장]  [취소]            │
│  ▶ 배관  (depth 1) [+ 추가] │                                      │
│  ▶ 전기  (depth 1) [+ 추가] │  ⚠ 하위 기자재가 있는 분류는       │
│                              │     비활성화만 가능 (삭제 불가)      │
│  [+ 최상위 분류 추가]        │                                      │
└──────────────────────────────┴──────────────────────────────────────┘
```

| 요소 | 연결 | 설명 |
|---|---|---|
| 분류 트리 | equipment_categories (depth 1→2→3→4) | 클릭 시 우측에 편집 폼 표시 |
| + 추가 버튼 | 해당 depth+1 신규 분류 생성 | depth 4 이하 추가 불가 |
| ✎ 수정 | category_name, sort_order 수정 | |
| 🗑 삭제/비활성 | 하위 기자재 없으면 삭제, 있으면 비활성만 가능 | equipments COUNT 확인 후 분기 |
| 정렬순서 | sort_order | 같은 depth 내 화면 노출 순서 |
| 코드 | category_code | Excel 업로드 시 기자재-분류 연결 기준 |
