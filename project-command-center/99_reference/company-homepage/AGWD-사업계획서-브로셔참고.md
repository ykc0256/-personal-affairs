## 1. Problem — 수처리 설계 영역의 공백 (브로셔 1Page 핵심)

### 산업 현황
- 전 세계 수처리 AI 적용은 대부분 **운영(O&M) 단계에 집중** (Veolia, SUEZ, Xylem 등)
- **설계 단계는 AI 적용이 가장 지연된 공백 영역** — 공정 선택, 설비 구성, 용량 산정, 배치 설계가 여전히 엔지니어 개인 경험과 수작업에 의존
- LLM 래핑·UX 재포장형 시도들은 이미 시장에서 성과 없이 정리 중

### 구조적 위기
- AEC 산업(설계·시공·운영) 전반의 엔지니어 **고령화 + 신규 인력 감소**
- 설계–시공–운영이 **단절된 구조**에서 수행 → 반복 작업·비효율 누적
- **설계 단계 결정이 시설 수명 20~30년간 에너지 소비와 운영비 결정** (US EPA: 에너지 비용이 운영비의 25~40%)
- 설계 단계 AI 적용 = 단순 효율 개선이 아닌 **산업 구조 자체를 바꾸는 전략적 전환점**

### 기술적 장벽
- GPS-X, BioWin 등 시뮬레이션 프로그램은 성능 검증용일 뿐, **실제 설계 산출물(도면·BOM·CAPEX·3D)로 자동 전환 불가**
- 시뮬레이션 → 실제 설계를 잇는 **미들웨어(Middleware)가 존재하지 않았던 것**이 근본 장벽

## 2. Solution — GWD (브로셔 핵심 엔진 소개)

### GWD 정의
> **GWD(Generative Water Design)** — 수처리 도메인 지식과 설계 로직이 내재화된 엔지니어링 엔진. 시뮬레이션 결과를 실제 설계 산출물로 전환하는 **세계 유일의 수처리 상세설계 미들웨어.**

### GWD 핵심 기능 (4단계 통합 흐름)
```
프로젝트 생성
→ 공정계획 (물질수지·공정용량 계산, 구조물/기계/전기 규격 자동 선정)
→ 3D 레이아웃 (구조물·설비·배관 라우팅·전기 트레이 연결 자동 배치)
→ 설계 산출물 생성 (PFD/P&ID/전기도면/평면도/단면도/ISO/물량/CAPEX·OPEX)
```

### 차별점 — 기존 방식 vs GWD
| 구분 | 기존 방식 | GWD 방식 |
|---|---|---|
| 공정-3D-도면 연결 | 각각 개별 도구에서 분리 수행 | 단일 플랫폼에서 통합 자동 수행 |
| 설계 변경 시 | 전 산출물 반복 수정 | 공정 조건 변경 시 전체 자동 갱신 |
| 산출물 일관성 | 산출물 간 불일치 발생 | 공정 계산 결과가 후속 단계에 자동 반영 |
| 설계 기간 | 6개월~1년 (상세설계) | **수 시간 내 해결** |

### Proof Points (홈페이지·브로셔 신뢰 근거)
| 항목 | 내용 |
|---|---|
| 개발 기간 | **2018년부터 약 8년** |
| 자체 투자 | **50억 원 이상** |
| 기술성숙도 | **TRL 5** — 사내 실제 설계 업무 적용 |
| 설계 기간 단축 | **기존 대비 80% 단축** 목표 (Transcend 96% 단축 대비 보수적 목표) |
| 설계 비용 절감 | **기존 대비 50% 절감** 목표 |
| 특허 | 국내(KR 10-2344936), **미국(US 12518327 B2)**, 인도(IN 538341), 중국(등록예정), 유럽(심사중) — **5개국** |
| 글로벌 최초 | **전 세계적으로 수처리 상세설계까지 End-to-End 자동화 가능한 유일한 시스템** |

## 3. AGWD — GWD의 다음 단계 (WAI 브로셔 핵심)

### AGWD 정의
> **AGWD(AI-Based Generative Water Design)** — GWD 설계 엔진 위에 도메인 특화 AI 의사결정 계층을 결합한 하이브리드 설계 시스템. **엔비디아 CUDA처럼 수처리 설계 전체에서 쓰이는 플랫폼**이 최종 목표.

### GWD → AGWD 구조적 전환
- **GWD**: 설계 실행(Execution) 자동화 — 공정 계산, 설비 구성, 3D, 산출물 생성
- **AGWD**: 설계 **의사결정(Decision)** 지원 — 다중 대안 생성, 최적안 비교, AI 검증

### 6개 AI 모듈
| 모듈 | 기능 |
|---|---|
| ① AI Design Assistant | 자연어 기반 설계 입력 및 조건 구조화 |
| ② Mass Balance AI | Rule+ML 하이브리드 공정 성능 예측 (정확도 92~97% 목표) |
| ③ Design Verification AI | 4단계 설계 검증 — 규정/설비정합성/3D충돌/경험기반 오류 탐지 |
| ④ 3D Layout AI | 설비 배치·배관 라우팅 자동 최적화 (부지 활용률 20%↑, 배관 15%↓) |
| ⑤ 탄소배출 산정 AI | Scope 1~3 및 내재 탄소 자동 산정, 설계 대안 간 비교 |
| ⑥ Tacit Knowledge Engine | 30년 설계 경험·암묵지를 OCP Ontology로 구조화 → RAG 기반 활용 |

### AGWD 발전 로드맵
```
1단계 (본 과제, 18개월):
  부강테크 보유 공법(아나목스·MBR·A2O 등) 기반
  환류 루프 완성 + 상용화 (TRL 5 → TRL 8)

2단계 (플랫폼 확장):
  전 세계 공정 기술로 확장
  ML 기반 머신러닝 학습 → 글로벌 SaaS 플랫폼
```

## 4. 환류 구조 — WATER AI CYCLE (홈페이지 2Page 핵심)

```
설계(AGWD) → 조달(Procurement) → 시공(Construction)
    ↑                                        ↓
재설계(Re-Design) ← 운영 데이터 ← 운영(Operation)
```

- **부강테크가 설계·시공·운영 전 주기를 자체 수행** → 운영 데이터를 직접 확보해 설계 기준으로 환류할 수 있는 **유일한 구조**
- 1단계: 부강테크 보유 공법 대상 환류 루프 완성
- 2단계: 전 세계 공정 기술로 환류 범위 확장

## 5. 시장 및 경쟁 포지셔닝

### 직접 경쟁사
| 경쟁사 | 비교 포인트 |
|---|---|
| **Transcend (TDG)** | 수처리 도메인 자동설계 분야 사실상 유일한 직접 경쟁사. 25일→1일(96% 단축) |
| ESAIN/ESAPRO | 상세설계 CAD 도구. 공정 계산 자동화 없음 — **직접 경쟁 아님** |
| GPS-X, BioWin | 시뮬레이션 프로그램. 설계 산출물로 자동 전환 불가 — **연동 대상** |
| Aspen OptiPlant | CAPEX 30% 절감 실증. 수처리 특화 아님 |

### GWD 차별점 vs Transcend
- **GWD**: 공정 계산 + 3D 레이아웃 + 도면 + 물량 + CAPEX/OPEX — **End-to-End 통합**
- **Transcend**: 공정 설계 중심. 3D 설계까지 포함하지 않음
- AGWD 목표: 설계 기간 **80% 단축** (Transcend 96% 단축 대비 보수적 설정)

## 7. 브로셔/홈페이지 메시지 활용 포인트

### GWD 브로셔에 쓸 수 있는 표현
- **"AI 시대, 수처리 설계 영역의 공백을 메우는 엔지니어링 엔진"**
- **"시뮬레이션 결과를 실제 설계 산출물로 전환하는 세계 유일의 미들웨어"**
- **"8년·50억 원 자체 투자, TRL 5, 5개국 특허"**
- **"설계 기간 최대 80% 단축, 설계 비용 50% 절감"**
- **"공정 계산부터 3D 레이아웃, 도면, 물량, CAPEX/OPEX까지 단일 플랫폼"**

### WAI/AGWD 브로셔에 쓸 수 있는 표현
- **"GWD 위에 AI 판단 계층을 더한 AGWD — 설계 실행을 넘어 설계 의사결정까지"**
- **"설계→조달→시공→운영→재설계로 이어지는 환류 구조"**
- **"부강테크는 EPC 전 주기를 자체 수행해 운영 데이터를 설계로 환류할 수 있는 유일한 구조"**
- **"수처리 설계 AI의 진짜 장벽은 데이터 부족이 아니라 기반 엔진의 부재였다"**

### 주의할 표현 (사업계획서 특성상 과장 있음, 홈페이지에서 조절 필요)
- TRL 8, 18개월 상용화, 100억 매출 등 **과제 목표 수치는 홈페이지 전면에 사용 금지**
- "세계 최초·최고" 등 강한 표현은 **근거 명시 필수**
- AGWD의 AI 모듈은 현재 개발 중 — **"개발 완료"로 표현 금지**
