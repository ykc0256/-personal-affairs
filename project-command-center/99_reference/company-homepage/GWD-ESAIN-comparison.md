# GWD vs ESAIN 솔루션 비교 정리

- 원본 파일: `GWD-ESAIN 솔루션 비교.pdf`
- 작성일: 2026-05-14
- 목적: GWD와 ESAIN의 역할 차이를 명확히 하고, 직접 경쟁 여부 및 브로셔·홈페이지 메시지에 활용할 포지셔닝 근거 파악

---

## 핵심 결론

| 구분 | GWD | ESAIN / ESAPRO |
|---|---|---|
| 제품 정의 | 수처리 플랜트 설계 자동화 프로그램 | CAD 기반 산업 플랜트 상세설계 소프트웨어 |
| 자동화 위치 | **기본설계 / 공정계산 자동화** (30% Design 수준) | **상세설계 / 문서·물량 산출 자동화** |
| 핵심 역할 | 설계조건 입력 → 공정 계산 → 구조물/기계 규격 선정 → PFD/P&ID/BOM 생성 | 사용자가 작성한 P&ID/3D 모델 → ISO/자재산출서/BOM/배관 제작도 생성 |
| 사용자 역할 | 조건 입력 후 자동 산출물 검토·보정 | CAD 모델을 직접 만들고 산출물 자동화 활용 |
| 플랫폼 구조 | Unity 기반 PC 설치형 (자체 환경) | AutoCAD/BricsCAD 기반 CAD 확장형 |
| 핵심 분야 | 수처리 플랜트 설계 | 산업 플랜트 설계 전반 (Water, Biogas, Hydrogen, Oil&Gas 등) |

**결론: ESAIN은 GWD의 직접 경쟁사가 아님. 자동화 위치가 다름 (기본설계 vs 상세설계)**

---

## 설계 단계별 위치

```
[설계조건 입력]
       ↓
  ← GWD 영역 →
공정 계산 / 물질수지 / 구조물·장비 sizing
PFD / P&ID 자동 생성 / 3D Layout / BOM (30% Design)
       ↓
  ← ESAIN 영역 →  (향후 연계 가능)
사용자 작성 P&ID → 3D Piping → ISO / 자재산출서 / BOM / 배관 제작도
(상세설계 / 제작 물량 성격)
```

---

## GWD 핵심 강점 (브로셔/홈페이지 활용 포인트)

- **공정 조건 → 설계 산출물 전 과정이 하나의 자체 환경에서 완결**됨 (ESAIN은 AutoCAD 종속)
- 공정 계산, 구조물/기계/전기/계측 규격 자동 선정이 핵심 (ESAIN은 이 영역 없음)
- 설계변경 시 전체 산출물이 함께 갱신되는 구조 (반복작업 최소화)
- Python 기반 계산 엔진 + Unity 기반 3D + SVG/DXF 도면 생성 — 독자 기술 스택

---

## ESAIN과의 관계 정리

| 구분 | 내용 |
|---|---|
| 직접 경쟁 여부 | 아님. P&ID/3D/BOM 영역에서 일부 접점이 있으나 자동화 위치가 다름 |
| 위협 수준 | 공정 선정/sizing 영역은 낮음. 수처리 고객군/P&ID/3D 영역은 중간 |
| 활용 가능성 | GWD 기본설계 결과 → ESAIN으로 상세설계 연계하는 워크플로우 검토 가능 |
| 모니터링 필요 | GWD가 향후 상세설계 영역으로 확장 시 ESAIN의 P&ID-3D-자재산출 연계 방식 참고 |

---

## 경쟁 위협 평가 요약

| 위협 영역 | 위협 수준 |
|---|---|
| 공정 선정/sizing | **낮음** — ESAIN 핵심 기능 아님 |
| 수처리 고객군 | 중간 — case history에 Water & Waste Treatment 4건 존재 |
| P&ID | 중간 — GWD는 자동 생성, ESAIN은 CAD 기반 작성/관리 |
| 3D 모델 | 중간 — GWD는 layout, ESAIN은 상세 3D piping |
| BOM/BOQ | 중간 — 둘 다 물량 산출 제공하나 상세도와 목적이 다름 |
| Autodesk Plant 3D와 기능 중복 | **높음** — ESAIN의 실제 경쟁자는 Autodesk Plant 3D |

---

## 브로셔/홈페이지 메시지에 활용할 포인트

1. **GWD는 기본설계 자동화 엔진** — 경쟁사(ESAIN 포함)가 사람이 직접 CAD를 그려야 하는 것과 달리, GWD는 조건 입력만으로 설계 산출물을 자동 생성
2. **자체 완결 플랫폼** — AutoCAD/BricsCAD 같은 외부 CAD에 종속되지 않는 독자 설계 환경
3. **수처리 특화** — ESAIN이 Oil&Gas, Biogas 등 범용인 반면, GWD는 수처리 공정 계산과 부강테크 공법 데이터를 내장한 도메인 특화 엔진
4. **AGWD 확장 방향** — GWD 기본설계 → 상세설계 → 운영까지 이어지는 플랫폼으로 발전 시, ESAIN이 커버하는 상세설계 영역도 자체 흡수하거나 연계하는 구조 가능
