# 260120 개인사업계획서 정리

원본: `99_reference/vendor-item-db/originals/260120_개인사업계획서_최윤규.xlsx`

## 문서 성격

2026년 개인 OKR 및 분기별 업무 계획 문서입니다. Vendor DB와 Item DB 설계는 `Procurement 데이터 구조 1건 구축` 항목에 포함되어 있습니다.

## 관련 OKR

| 구분 | 내용 |
| --- | --- |
| OBJ | Procurement 데이터 구조 1건 구축 |
| KR | Vendor 관리 DB Table 컬럼 정의 2건 |
| 우선순위 | 4 |
| 연간 목표 | Vendor DB, Item DB 컬럼 설계 및 구조화, GWD 관리자 페이지 반영 |

## 분기 계획

| 분기 | 계획 | 현재 해석 |
| --- | --- | --- |
| 1분기 | Vendor DB Table 컬럼 설계 및 구조화 | 계획 대비 미진행. 즉시 착수 필요 |
| 2분기 | 아이템 DB Table 컬럼 설계 및 구조화 | 2026-05-14 기준 미착수. 2분기 내 완료하려면 일정 재설계 필요 |
| 3분기 | GWD 관리자 페이지 반영 | DB 구조 확정 후 개발 반영 범위 정의 필요 |
| 4분기 | 별도 계획 없음 | 운영 안정화 또는 개선 항목으로 관리 가능 |

## 원문에서 확인한 세부 계획

- Vendor DB 관련 프로그램 내 반영 계획 수립
- Vendor DB와 GWD 간의 연계성 검토
- Vendor DB Table 설계 및 구조화
- GWD 관리자 페이지 내 Vendor DB 반영

## 프로젝트 반영 사항

- Vendor DB와 Item DB를 별도 테이블로 설계하되, GWD 관리자 페이지와 연결될 수 있는 관계 구조가 필요합니다.
- Vendor DB 설계 지연은 공식 리스크로 관리합니다.
- Item DB는 2분기 업무로 남아 있으므로 2026-06-30 이전 최소 설계안 확정이 필요합니다.
