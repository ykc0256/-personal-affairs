Vendor DB (P팀 논의)
진행상황
Vendor 평가 진행중, 아이템별 DB 진행중
아이템별 DB는 기자재별 설계 스펙을 웹페이지에서 입력 후 저장
3분기에 실제 사용 계획함
논의내용
(P팀)데이터베이스 대체 사용으로 우선 MS365 사용 중이나 data 정리가 어려움
(P팀)현재 시점에서 GWD 연계는 실질적으로 어려워 보임
어려운 이유는 사용 목적임
TSK에서의 사용 목적은 실행검토, 기자재 spec 별 datasheet정리, 프로젝트별 기자재 DATA 정리
PS에서 설계를 하고 TSK에 넘어오는 구조 이기 때문에 GWD를 PS에서 사용을 해야 DB를 같이 사용하는 접점이 생길 듯
GWD 사용 목적 정리 필요 (기타, 민수/TK)
P팀은 민수/TK 에 대한 SPEC 정리중
DB를 같이 정리하기에는 EPC팀에서 정리하고 있는 내용을 WAI팀에서 파악이 힘들고 EPC팀도 처음 정리하는 것이기 때문에 컬럼(내용)이 계속적으로 바뀌기 때문에 같이 정리하기는 힘들듯
결론
현재 상황에서 DATA 연계는 사용 목적이 달라 같이 진행하기는 어려움
DATA 사용 목적 정리 필요(TK/민수)
WAI팀에서는 DATA 등록 시스템 구조를 제시해줬으면 함
P팀에서의 DB사용 이유는 협업하는 목적(MS365 로는 협업이 힘들다)
DB 사용법, 시스템 구조, 데이터 업로드 템플릿 등
