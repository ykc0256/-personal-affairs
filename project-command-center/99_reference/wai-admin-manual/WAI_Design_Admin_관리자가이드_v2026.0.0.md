## 개요

**WAI Design**은 수처리 플랜트 엔지니어링 구성을 위한 통합 프로그램입니다.  
공정 계산 → 구조물/기계 배치 → 배관/전기 구성 → 도면 생성 → 수량 산출까지 전 과정을 단일 환경에서 처리합니다.

**WAI Design Admin**은 시스템 운영 및 설계 기준 관리를 위한 관리자 웹페이지입니다.

## 4. 코드 관리

코드 계층: **Level 1(코드 그룹) → Level 2(대분류) → Level 3(중분류) → Level 4(소분류)**

### 코드 등록 절차

1. 표준 코드 파일에서 등록할 코드의 레벨 확인
2. 상위 카테고리까지 선택 후 **등록** 버튼 클릭 (Level 별 상위 선택 기준 상이)
3. 코드·국문 코드명·영문 코드명·순서·최하위 노드 여부 입력 후 저장

> **중요:** 공정 코드 등록 시 최하위 코드는 반드시 **최하위 노드 = Y** 로 설정해야 WAI Design 공정 패널에서 정상 표출됩니다.

### 코드 다건 등록 (Level 2~4)

1. 대상 레벨의 parent code까지 카테고리 선택 → **다건 등록** 버튼
2. **엑셀 양식 다운로드** → `코드 등록 양식.xlsx` 작성 → **Excel 파일 가져오기** → 저장

## 6. 유입 종류 관리

### 유입 탭 — 유입 항목 추가

1. **신규 추가** → 유입 항목명(국문) 드롭다운 선택 (영문 자동 입력)
2. 심볼 색상 설정, 심볼 파일(.svg) 업로드
3. Metric / USCS 섹션별로 **계산식 파일** 업로드
4. 계산식 등록 완료 시 파라미터 값 표시 확인 후 저장

> 원하는 유입 항목명이 없으면 코드 관리 메뉴에서 먼저 코드 등록 필요

### 파라미터 관리

- **파라미터 관리** 버튼 → 수질 파라미터 추가·수정·삭제

### 유출 탭

유입 탭과 동일한 방식으로 유출 항목·계산식·파라미터 관리

## 7. 공정 관리

### 7.1 계산식

- 공정 등록 시 **계산식은 필수 항목**
- 계산식 파일 형식: **Python (.py)**
- 유효성 검사 항목:
  - `unit_system = UnitSystemType.METRIC/USCS` 정의 여부
  - Python 기본 문법
- 등록된 계산식이 프로젝트에서 사용 중이면 삭제 불가 (해당 프로젝트 삭제 후 삭제 가능)

### 7.2 공정 카드

- 구조물 수·기계 수·POC 정보에 따라 여러 개 등록 가능
- DWG → SVG 변환 필요 (SVG Converter 사용, [부록2] 참조)
- SVG 파일 + 원본 DWG 파일 함께 업로드

### 7.3 매핑 P&ID

- 공정 카드별 하위에 여러 P&ID 등록 가능
- **매핑 Excel**: 필수 항목
- PID 파일 작성 규칙:
  - 블록 처리 필수
  - 좌표계 원점에 배치
  - 멀티 도면 파일명: `M_` 접두어 필수

### 7.4 용량 계산서

- 용량 계산서 템플릿(.xlsx) 업로드
- 계산식이 등록된 경우에만 파일 선택·등록 버튼 활성화
- DATA-IN SHEET 명칭 형태: `{공정명}_DATA_I`
- LOOK UP 범위: 최대한 넓게 설정
- 참조 항목명: 공정계산식의 MAP 파트 정의명과 일치 필요

## 8. 공종 관리

공종 관리 탭에서 **구조물·기계·배관 자재·전기 자재·계측기·3D Asset** 품목 등록·관리

### 공통 등록 패턴

| 탭 | 등록 방식 |
|---|---|
| 구조물 | 계산식·DTD·썸네일·Revit 파일을 ZIP으로 압축 후 업로드 |
| 기계 | Excel 양식 다운로드 → 작성 후 업로드, 3D/Revit/썸네일 별도 등록 |
| 배관 자재 | 기계 탭과 동일 |
| 전기 자재 | Excel 양식 다운로드 → 작성 후 업로드 |
| 계측기 | Excel 양식 다운로드 → 작성 후 업로드 |
| 3D Asset | 3D 라이브러리 또는 3D 프리셋 탭에서 등록 |

### 8.1 구조물 계산식 유효성 검사

- `unit_system = UnitSystemType.METRIC/USCS` 정의 여부
- 규격 계산식·검증 계산식·수리 계산식 존재 여부

### 8.2 기계 탭 주요 규칙

- 대분류·중분류 → 기계 유형 순서로 카테고리 선택
- 계산식을 **중분류에 매칭**: 중분류 전체에 동일 계산식 적용
- 계산식을 **기계 타입에 매칭**: 기계 타입별 다른 계산식 적용 가능
- 기계 계산식 버전 관리: 항상 최신 버전 우선 적용
- 모델 파일명에 **기계 코드 포함 필수** (파일명으로 코드 식별)
- 동적 변형 장치 등록: [부록3] Submesh 참조

### 8.2 기계 계산식 유효성 검사

- `unit_system = UnitSystemType.METRIC/USCS` 정의 여부
- 규격 계산식 존재 여부

## 부록

### [부록1] DTDConverter

**목적:** Revit에서 생성된 `.rvt` / `.rfa` 파일을 WAI Design용 `.dtdx` 형식으로 변환

> **지원 버전:** Revit 2023 기준. 이후 버전은 별도 지원 없음.

**설치 순서:**
1. 설치 패키지 실행 → Next → I Agree → 설치 진행 → Finish
2. 제어판 > 프로그램 및 기능에 `DTDConverter 2026.x.xx` 항목 확인

**Revit에서 사용:**
1. Revit 실행 → 팝업 메시지에서 **항상 로드** 또는 **한 번 로드** 선택
2. Revit 애드인 탭에 DTDX 관련 항목 표시됨
3. 작업 완료 후 **Family 내보내기** 아이콘 클릭 → 요소 선택 → 저장 경로 지정 → 확인

**제거:** 제어판 > 프로그램 및 기능 > `DTDConverter 2026.x.xx` 마우스 우클릭 > 제거/변경

---

### [부록2] SVGConverter

**목적:** DWG 도면을 SVG 파일로 변환  
(Admin에서 P&ID·PFD·범례 도면 등록 및 WAI Design 앱에서 부지 도면 업로드 시 SVG만 지원)

**설치 순서:**
1. 설치 파일 실행 → Next → 설치 경로 지정 → Install → Finish

**파일 변환:**
1. SVGConverter 실행
2. **Input Folder**: 변환 대상 DWG 파일 포함 폴더 지정
3. **Output Folder**: 변환 결과 SVG 파일 저장 폴더 지정
4. **Convert** 버튼 클릭 → 완료 후 OK → 출력 폴더에서 SVG 파일 확인

---

### [부록3] Submesh 생성/업로드

**목적:** 동적 변형 장치(길이가 가변적인 3D 모델) 등록  
CONSTRAINT(고정 부분) + SCALE(가변 부분)로 분리하여 위치 이동 시 SCALE 영역 자동 조절

**DTDSquare로 Sub-mesh 생성:**
1. DTDSquare 앱 실행 → 파일 열기 → `.dtdx` 파일 선택
2. 툴바 편집 도구 선택 → 직교모드(Ortho) 전환
3. 복수 오브젝트인 경우: `Ctrl+A` 전체 선택 → 병합 도구로 단일 오브젝트화
4. 슬라이스 도구로 CONSTRAINT/SCALE 영역 분리
5. 각 오브젝트 우클릭 > 속성 추가 → 그룹이름·키이름 입력, 값: `CONSTRAIN` 또는 `SCALE`
6. Part Type에서 속성 확인 → 상단 메뉴 저장

**관리자 웹에서 Submesh 등록:**
1. Admin > 공종 관리 → 해당 탭에서 대상 항목 상세정보 보기
2. 수정 버튼 클릭 → 3D 항목에 DTDSquare 생성 파일 업로드

---

### [부록4] Revit 복합 패밀리 색상 파라미터 연동

**목적:** WAI Design에서 구조물/기기 색상을 Revit RGB 값으로 표시  
최상위~최하위 패밀리의 모든 오브젝트 색상을 하나의 파라미터로 연동 → 일괄 색상 변경 가능

**연동 절차 (3단계 복합 패밀리 예시):**
1. 최상위 패밀리의 객체를 색상 파라미터와 연동 (파라미터는 **Instance** 타입으로 생성)
2. 하위 복합 패밀리 더블 클릭 → 해당 편집기 내 객체를 색상 파라미터와 연동
3. 최하위 패밀리까지 반복 → **Load into project** / **Load into project and close**로 상위에 재로드
4. 각 단계에서 하위 패밀리를 상위 색상 파라미터와 매핑(연동)
5. 전체 구성품에 반복 적용 → 최상위 파라미터 하나로 전체 색상 제어 가능

> 단계가 많아도 동일 방식으로 최하위까지 모두 연동하면 됩니다.
