# Project Command Center

여러 프로젝트를 한 곳에서 관리하기 위한 통합 운영 레포입니다. 핵심 기준은 일정입니다.

## 운영 원칙

1. 모든 프로젝트는 `01_calendar/master-schedule.md`에 일정이 먼저 등록되어야 합니다.
2. 프로젝트별 상세 내용은 `02_projects/` 아래에 둡니다.
3. 주간 회고와 다음 주 계획은 `03_weekly-reviews/`에 남깁니다.
4. 임시 메모, 갑자기 들어온 요청, 아직 분류하지 않은 자료는 `00_inbox/`에 둡니다.
5. 완료되었거나 중단된 프로젝트는 `05_archive/`로 이동합니다.

## 폴더 구조

```text
00_inbox/           분류 전 메모, 요청, 아이디어
01_calendar/        전체 일정, 마일스톤, 마감 관리
02_projects/        프로젝트별 실행 문서
03_weekly-reviews/  주간 리뷰와 다음 주 계획
04_templates/       반복 작성용 템플릿
05_archive/         완료/보류/중단 프로젝트
99_reference/       공통 참고자료, 규칙, 링크
```

## 기본 운영 흐름

1. 새 프로젝트가 생기면 `04_templates/project-template.md`를 복사해 `02_projects/프로젝트명.md`를 만듭니다.
2. 주요 마감, 검토일, 납품일을 `01_calendar/master-schedule.md`에 등록합니다.
3. 매주 금요일 또는 월요일에 `04_templates/weekly-review-template.md`를 복사해 주간 리뷰를 작성합니다.
4. 일정 지연, 의사결정 대기, 외부 의존성은 프로젝트 문서의 `리스크 / 블로커`에 기록합니다.

## 상태 기준

| 상태 | 의미 |
| --- | --- |
| 예정 | 아직 착수 전 |
| 진행 | 현재 작업 중 |
| 대기 | 외부 답변, 승인, 자료 수신 대기 |
| 위험 | 일정 지연 가능성이 높음 |
| 완료 | 산출물 제출 또는 내부 완료 |
| 보류 | 일정 미정 또는 우선순위 하락 |
