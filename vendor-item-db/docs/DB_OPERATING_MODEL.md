# DB 연동 운영 구조 설계

## 목표

관리자가 화면에서 분류, 기자재, 업체, 업체-기자재 연결, 가격, 평가 기준을 관리하면 그 변경이 전체 서비스 구조에 즉시 반영되는 형태를 목표로 합니다. 화면은 임시 입력 폼이 아니라 DB 마스터를 관리하는 도구여야 합니다.

핵심 원칙:

- DB 마스터가 단일 기준 정보입니다.
- 조회 화면은 마스터 데이터를 그대로 반영합니다.
- 삭제는 기본적으로 물리 삭제가 아니라 비활성화입니다.
- 코드값과 분류 체계는 별도 관리 화면에서 관리합니다.
- Excel 업로드와 수동 입력은 같은 검증 규칙을 사용해야 합니다.
- 관리자가 구조를 바꾸면 기자재, 업체, 가격, 평가, 대시보드에 일관되게 반영되어야 합니다.

## 현재 구조 평가

현재 구현은 DB에 직접 연결되어 있지만, 운영 구조 관점에서는 아직 다음 부분이 약합니다.

| 영역 | 현재 상태 | 보완 필요 |
| --- | --- | --- |
| 기자재 분류 | `equipment_categories`를 select로 선택 | 분류 코드, 경로, 활성 상태, 하위 분류 관리 필요 |
| 기자재 코드 | 수동 입력 | 분류 코드 기반 자동 제안 또는 중복 검증 강화 필요 |
| 기자재 관리 | 추가/수정/비활성화 가능 | 분류 체계와 코드 정책 연결 필요 |
| 업체 관리 | 조회 중심 | 업체 추가/수정/비활성화, 업체 코드 관리 필요 |
| 업체-기자재 연결 | 가격 추가 시 일부 자동 생성 | 별도 연결 관리 화면 필요 |
| 가격 관리 | 기자재 상세 행에서 가능 | 업체-기자재 연결 상태와 검증 규칙 강화 필요 |
| 평가 관리 | 조회 중심 | 평가 기준 관리와 평가 입력 UI 필요 |
| Excel 업로드 | 예정 | 마스터 코드 기준 검증, 오류 행 수정/재처리 필요 |

## 권장 마스터 관리 구조

### 1. 분류 마스터

대상 테이블: `equipment_categories`

관리 화면:

- 분류 트리 조회
- 분류 코드 확인
- 상위 분류 변경
- 분류명 수정
- 정렬 순서 수정
- 활성/비활성 처리
- 하위 분류 추가

운영 규칙:

- 기자재 추가/수정 화면에서는 분류명만 보여주지 않고 전체 경로와 분류 코드를 함께 표시합니다.
- 비활성 분류는 신규 기자재 등록 시 선택할 수 없습니다.
- 기존 기자재가 연결된 분류는 물리 삭제하지 않습니다.
- 분류 코드 변경 시 기자재 코드 자동 생성 규칙에 영향이 있는지 확인해야 합니다.

예시 표시:

```text
선택 분류: 전기 > 제어반 > PLC 패널
분류 코드: ELEC-CTRL-PLC
```

### 2. 기자재 마스터

대상 테이블: `equipments`

관리 위치:

- 기자재 조회 화면 상단: 신규 추가
- 기자재 행 상세 영역: 수정, 비활성화
- 추후 관리 화면: 대량 검토, 비활성 기자재 복구

운영 규칙:

- `equipment_code`는 필수이며 unique입니다.
- `gwd_equipment_id`는 선택값이지만 입력된 경우 unique입니다.
- 기자재 삭제는 `is_active=false`로 처리합니다.
- 기자재 추가/수정 시 분류 경로와 분류 코드를 명확히 보여줍니다.
- 기자재 코드 자동 생성은 분류 코드 정책이 확정된 뒤 도입합니다.

권장 보완:

- 분류 선택 UI를 단순 select에서 트리 선택 또는 검색형 선택으로 변경
- 선택한 분류의 `category_code`, 전체 경로 표시
- 기자재 코드 중복 검증 메시지 강화
- 비활성 기자재를 별도 필터로 조회할 수 있게 확장

### 3. 업체 마스터

대상 테이블: `vendors`

관리 위치:

- 업체 조회 화면 상단: 신규 추가
- 업체 상세 또는 행 상세 영역: 수정, 비활성화
- 관리 화면: 업체 코드/외부 GWD 코드 검토

운영 규칙:

- `vendor_code`는 필수이며 unique입니다.
- `gwd_vendor_id`는 선택값이지만 입력된 경우 unique입니다.
- 업체 삭제는 `is_active=false`로 처리합니다.
- 비활성 업체는 신규 가격 등록과 신규 업체-기자재 연결에서 선택할 수 없습니다.

권장 보완:

- 업체 코드, GWD ID, 업체 유형, 국가, 재무 등급을 관리 가능한 필드로 제공
- 연락처 `contact_info`는 JSON 구조를 화면에서 사용하기 쉬운 필드로 분해해서 입력
- 업체 상세에서 취급 기자재와 최근 가격/평가를 함께 표시

### 4. 업체-기자재 연결

대상 테이블: `vendor_items`

관리 위치:

- 기자재 상세 행: 해당 기자재의 취급 업체 연결 관리
- 업체 상세 화면: 해당 업체의 취급 기자재 연결 관리

운영 규칙:

- `vendor_id`, `equipment_id` 조합은 unique입니다.
- 연결 삭제는 `is_active=false`로 처리합니다.
- 가격 이력은 `vendor_items` 관계를 전제로 저장됩니다.
- 가격 추가 중 연결이 없으면 자동 생성할 수 있지만, 장기적으로는 연결 관리 UI에서 명시적으로 관리하는 것이 좋습니다.

권장 보완:

- 기자재 상세 행에 “업체 연결 추가” 제공
- 업체 상세에 “취급 기자재 추가” 제공
- 연결 비활성화 시 기존 가격 이력은 보존
- 비활성 연결은 신규 가격 입력 대상에서 제외

### 5. 가격 이력

대상 테이블:

- `design_prices`
- `execution_prices`

운영 규칙:

- 가격은 수정/삭제보다 이력 추가 방식이 기본입니다.
- 잘못 입력된 가격은 `is_voided=true`로 확정 제외합니다.
- 실행가는 `admin`, `procurement`만 조회/관리합니다.
- 가격 등록은 활성 기자재, 활성 업체, 활성 업체-기자재 연결에 대해서만 허용합니다.

권장 보완:

- 가격 등록 시 현재 최신 가격과 비교 표시
- 같은 기준일, 같은 업체, 같은 기자재의 중복 입력 경고
- 확정 제외 사유 입력 필드 추가
- Excel 업로드 가격도 동일한 검증 규칙 적용

### 6. 평가 기준 및 평가 이력

대상 테이블:

- `evaluation_criteria`
- `vendor_evaluations`
- `evaluation_scores`

관리 위치:

- 관리 화면: 평가 기준 관리
- 업체 상세 화면: 평가 등록/수정

운영 규칙:

- 평가 기준은 활성 기준만 신규 평가 입력에 사용합니다.
- 기존 평가 이력은 기준이 비활성화되어도 보존합니다.
- 총점과 등급은 기준 점수 기반으로 계산하거나, 계산 정책을 문서화한 뒤 저장합니다.

권장 보완:

- 평가 기준명, 가중치, 최대 점수, 정렬 순서 관리
- 업체 상세에서 평가 이력과 기준별 점수 조회
- 평가 등록 시 활성 기준 자동 로드

### 7. Excel 업로드

대상 테이블:

- `excel_uploads`
- `upload_error_rows`
- `attachments`
- 업로드 결과에 따라 각 마스터/이력 테이블

운영 규칙:

- 업로드는 화면 수동 입력과 같은 검증 규칙을 사용합니다.
- 업로드 파일의 분류 코드, 기자재 코드, 업체 코드는 DB 마스터 기준으로 검증합니다.
- 실패 행은 `upload_error_rows`에 저장하고 화면에서 수정/재처리할 수 있어야 합니다.
- 대량 등록은 즉시 반영보다 검토 후 반영 방식을 우선 검토합니다.

권장 흐름:

```text
파일 업로드
-> 컬럼 검증
-> 마스터 코드 검증
-> 중복/누락 검증
-> 미리보기
-> 반영
-> 성공/실패 결과 저장
```

## 화면 구조 권장안

```text
대시보드
  - 이상 항목 요약
  - 미등록 가격, 최근 변경, 업로드 실패

기자재
  - 기자재 조회
  - 기자재 추가/수정/비활성화
  - 업체 연결 관리
  - 가격 이력 관리

업체
  - 업체 조회
  - 업체 추가/수정/비활성화
  - 취급 기자재 연결 관리
  - 평가 이력 관리

가격
  - 가격 이력 통합 조회
  - 확정 제외 이력 조회
  - 업로드/수동 입력 이력 비교

평가
  - 평가 이력 통합 조회
  - 기준별 점수 조회

관리
  - 분류 관리
  - 평가 기준 관리
  - 사용자/권한 관리
  - Excel 업로드
  - 업로드 오류 행 관리
```

## 우선순위

### 1단계: 분류 체계 정리

- 분류 관리 화면 설계
- 기자재 추가/수정 폼에서 분류 코드와 경로 표시
- 분류 선택 UI 개선

### 2단계: 업체 화면 정리

- 업체 조회 화면 한글/레이아웃 복구
- 업체 마스터 추가/수정/비활성화 설계
- 업체 상세에서 취급 기자재와 평가/가격 연결 강화

### 3단계: 연결 관리

- 기자재 기준 업체 연결 관리
- 업체 기준 취급 기자재 연결 관리
- 연결 비활성화 정책 적용

### 4단계: 업로드 체계

- Excel 업로드 검증 규칙 문서화
- 업로드 미리보기와 오류 행 관리
- 수동 입력과 업로드의 공통 검증 로직 정리

### 5단계: 평가 기준 관리

- 평가 기준 관리 화면
- 업체 평가 등록/수정
- 총점/등급 계산 정책 정리

## 결론

현재 방향은 기자재 조회 화면에 운영 액션을 넣는 것이 맞지만, 분류와 코드 체계가 별도 마스터로 관리되지 않으면 장기적으로 데이터가 흔들릴 수 있습니다. 따라서 다음 구현은 업체 화면으로 바로 확장하기 전에 분류 코드/경로 표시와 분류 관리 설계를 먼저 반영하는 것이 좋습니다. 이후 업체 화면은 같은 원칙으로 업체 코드, 취급 기자재 연결, 평가 기준과 연결되도록 정리합니다.
