# 화면설계서

## 공통 방향

모든 화면은 DB 마스터를 기준으로 동작합니다. 관리자가 마스터 데이터를 수정하면 조회, 검색, 대시보드, 가격 관리, 평가 관리, Excel 업로드 검증에 같은 기준으로 반영되어야 합니다.

공통 원칙:

- 일반 사용자는 조회 중심입니다.
- `procurement`는 가격과 실행가 중심 권한을 가집니다.
- `admin`은 마스터 구조를 관리합니다.
- 삭제는 기본적으로 비활성화 처리입니다.
- 코드, 분류, 업체, 기자재, 평가 기준은 독립된 마스터로 관리합니다.

## 대시보드

목적:

- 운영자가 이상 항목을 빠르게 발견하고 관련 화면으로 이동합니다.

표시 항목:

- 평가 현황
- 가격 미등록 기자재
- 업로드 실패 요약
- 최근 가격 변경 이력

연동 기준:

- 활성 기자재와 활성 업체를 기준으로 집계합니다.
- 확정 제외된 가격은 최신 가격 계산에서 제외합니다.
- 업로드 실패는 `upload_error_rows` 기준으로 표시합니다.

## 기자재 화면

목적:

- 기자재 마스터와 가격/업체 연결 상태를 한 화면에서 확인하고 관리합니다.

주요 영역:

| 영역 | 설명 |
| --- | --- |
| 분류 트리 | `equipment_categories` 기준 계층 필터 |
| 검색 조건 | 기자재명, 모델명, 규격, 코드, 업체명, 국가 통합 검색 |
| 기자재 목록 | 분류, 모델/기자재명, 규격, 대표 업체, 국가, 가격, 평가 표시 |
| 행 상세 | 기자재 상태, 업체 연결, 가격 변경 이력 표시 |
| 기자재 관리 | `admin` 전용 추가/수정/비활성화 |
| 가격 관리 | `admin`, `procurement` 전용 가격 추가/확정 제외 |

기자재 추가/수정 입력 항목:

- 기자재 코드
- 분류
- 분류 코드 및 전체 경로 표시
- 기자재명
- 모델명
- 제조사 모델번호
- 단위
- GWD ID
- 규격/사양
- 비고

보완 필요:

- 분류 선택을 단순 select에서 트리/검색형 선택으로 개선
- 선택한 분류의 `category_code`와 경로 표시
- 기자재 코드 자동 제안 또는 코드 규칙 안내
- 비활성 기자재 조회 필터
- 업체-기자재 연결 관리 UI

## 업체 화면

목적:

- 업체 마스터와 취급 기자재, 평가, 가격 관계를 관리합니다.

주요 영역:

| 영역 | 설명 |
| --- | --- |
| 분류 트리 | 선택 분류의 기자재를 취급하는 업체 필터 |
| 검색 조건 | 업체명, 업체 코드, 취급 기자재명 검색 |
| 업체 목록 | 업체명, 유형, 국가, 재무 등급, 최근 평가 표시 |
| 업체 상세 | 업체 기본 정보, 취급 기자재, 평가 이력, 가격 이력 표시 |
| 업체 관리 | `admin` 전용 추가/수정/비활성화 |
| 평가 관리 | 권한 정책 확정 후 평가 등록/수정 |

업체 추가/수정 입력 후보:

- 업체 코드
- 업체명
- 업체 유형
- 국가
- 사업자번호
- 연락처
- 재무 등급
- 매출
- 자본금
- 기준 연도
- GWD 업체 ID
- 비고

보완 필요:

- 업체 화면 한글 깨짐 복구
- 업체 마스터 추가/수정/비활성화 설계
- 업체 상세에서 취급 기자재 연결 관리
- 업체 평가 등록/수정 UI

## 분류 관리 화면

목적:

- 기자재 분류 코드와 계층 구조를 관리합니다.

대상 테이블:

- `equipment_categories`

기능:

- 분류 트리 조회
- 분류 추가
- 분류명/코드 수정
- 상위 분류 변경
- 정렬 순서 수정
- 활성/비활성 처리

운영 규칙:

- 기존 기자재가 연결된 분류는 물리 삭제하지 않습니다.
- 비활성 분류는 신규 기자재 등록 시 선택할 수 없습니다.
- 분류 코드는 기자재 코드 자동 제안 정책의 기반이 될 수 있습니다.

## 업체-기자재 연결 관리

목적:

- 어떤 업체가 어떤 기자재를 취급하는지 명시적으로 관리합니다.

대상 테이블:

- `vendor_items`

진입 위치:

- 기자재 행 상세
- 업체 상세

기능:

- 업체 연결 추가
- 대리점명 입력
- 연결 활성/비활성 처리
- 연결된 업체 기준 가격 등록

운영 규칙:

- 같은 업체와 기자재 조합은 중복 등록할 수 없습니다.
- 연결 비활성화 후에도 과거 가격 이력은 보존합니다.

## 가격 화면

목적:

- 설계가/실행가 이력을 통합 조회하고 검증합니다.

대상 테이블:

- `design_prices`
- `execution_prices`

기능:

- 가격 이력 통합 조회
- 기자재/업체/분류 기준 필터
- 확정 제외 이력 조회
- 업로드 이력과 수동 입력 이력 구분

운영 규칙:

- 가격은 수정보다 신규 이력 추가가 기본입니다.
- 오류 데이터는 확정 제외로 처리합니다.
- 실행가는 권한 사용자만 조회합니다.

## 평가 화면

목적:

- 업체 평가 이력을 조회하고 기준별 평가를 관리합니다.

대상 테이블:

- `vendor_evaluations`
- `evaluation_scores`
- `evaluation_criteria`

보완 필요:

- 평가 기준 관리
- 업체별 평가 등록/수정
- 총점/등급 계산 정책 정리

## 관리 화면

목적:

- 마스터 구조와 운영 기준을 관리합니다.

하위 메뉴 후보:

- 분류 관리
- 평가 기준 관리
- 사용자/권한 관리
- Excel 업로드
- 업로드 오류 행 관리

관리 화면에서 바뀐 데이터는 각 조회 화면에 즉시 반영되어야 합니다.
